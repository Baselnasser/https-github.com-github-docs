**Advertencia:** Cuando cree flujos de trabajo y acciones, siempre debe considerar si el código podría ejecutar entradas no confiables de atacantes potenciales. Se tratará a algunos contextos como una entrada no confiable, ya que un atacante podrían insertar su propio contenido malintencionado. Para más información, vea "[Descripción del riesgo de las inyecciones de scripts](/actions/learn-github-actions/security-hardening-for-github-actions#understanding-the-risk-of-script-injections)".
