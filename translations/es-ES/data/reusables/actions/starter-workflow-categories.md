{% data variables.product.prodname_dotcom %} proporciona flujos de trabajo de inicio listos para usar para las siguientes categorías generales:
- **Implementación (CD)** . Para más información, vea "[Acerca de la implementación continua](/actions/deployment/about-deployments/about-continuous-deployment)". {% ifversion fpt or ghec %}: **Seguridad**. Para más información, vea "[Establecimiento de {% data variables.product.prodname_code_scanning %} mediante flujos de trabajo de inicio](/code-security/code-scanning/automatically-scanning-your-code-for-vulnerabilities-and-errors/setting-up-code-scanning-for-a-repository#setting-up-code-scanning-using-starter-workflows)".{% endif %}
- **Integración continua (CI)** . Para más información, vea "[Acerca de la integración continua](/actions/automating-builds-and-tests/about-continuous-integration)". 
- **Automatización**. Los flujos de trabajo de inicio de automatización ofrecen soluciones para automatizar flujos de trabajo, como evaluar las solicitudes de incorporación de cambios y aplicar una etiqueta basada en las rutas modificadas en la solicitud de incorporación de cambios, o bien saludar a los usuarios que colaboran por primera vez en el repositorio.
