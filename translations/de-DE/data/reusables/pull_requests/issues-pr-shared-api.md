Jeder Pull Request ist ein Issue, aber nicht jedes Issue ist ein Pull Request. Aus diesem Grund werden „freigegebene“ Aktionen für beide Features (z. B. das Verwalten von Zuweisungen, Bezeichnungen und Meilensteinen) in der Issues-API bereitgestellt.
