Les environnements sont utilisés pour décrire une cible de déploiement général comme `production`, `staging` ou `development`. Quand un workflow {% data variables.product.prodname_actions %} est déployé dans un environnement, l’environnement s’affiche dans la page principale du dépôt. Vous pouvez utiliser des environnements pour exiger l’approbation d’un travail, restreindre les branches pouvant déclencher un workflow ou limiter l’accès aux secrets. Pour plus d’informations sur la création d’environnements, consultez « [Utilisation d’environnements pour le déploiement](/actions/deployment/using-environments-for-deployment) ».
