Pour accéder aux ressources de chaque organisation sur {% data variables.product.product_name %}, le membre doit disposer d’une session SAML active dans son navigateur. Pour accéder aux ressources protégées de chaque organisation à l’aide de l’API et du Git, le membre doit utiliser un jeton d’accès personnel ou une clé SSH dont il a autorisé l’utilisation avec l’organisation. Les propriétaires d’entreprise peuvent à tout moment afficher et révoquer l’identité liée d’un membre, les sessions actives ou les informations d’identification autorisées.
