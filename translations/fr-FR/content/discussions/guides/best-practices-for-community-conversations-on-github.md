## Conversations de la communauté dans {% data variables.product.prodname_discussions %}

Comme {% data variables.product.prodname_discussions %} est un forum ouvert, vous pouvez collaborer sans code au dépôt d’un projet ou à une organisation, ainsi que recueillir plus rapidement divers commentaires et idées. Vous pouvez aider à conduire une conversation productive de plusieurs façons.

- Poser des questions pointues et des questions de suivi pour obtenir des commentaires spécifiques
- Capture d’une expérience diversifiée et distillation de celle-ci à des points principaux
- Ouverture d’un problème pour prendre des mesures en fonction de la conversation, le cas échéant

Pour plus d’informations sur l’ouverture d’un problème et sur le référencement croisé d’une discussion, consultez « [Ouverture d’un problème à partir d’un commentaire](/github/managing-your-work-on-github/opening-an-issue-from-a-comment) ».

## Apprentissage des conversations sur GitHub

Vous pouvez créer des discussions, des problèmes et des demandes de tirage, et y participer, en fonction du type de conversation que vous voulez avoir.

Vous pouvez utiliser {% data variables.product.prodname_discussions %} pour discuter d’idées générales, faire un brainstorming et développer les détails spécifiques d’un projet avant de le valider dans un problème, qui peut ensuite être délimité. {% data variables.product.prodname_discussions %} est utile pour les équipes dans un certain nombre de scénarios.
- Vous êtes dans la phase de découverte d’un projet et ne savez pas encore dans quelle direction votre équipe veut aller
- Vous voulez recueillir les commentaires d’une communauté plus large sur un projet
- Vous voulez séparer les correctifs de bogues, les demandes de fonctionnalités et les conversations générales
- Vous souhaitez mesurer l’intérêt pour de nouvelles idées et jauger l’opinion de la communauté à l’aide de sondages

Vous pouvez utiliser des discussions concernant de dépôt pour discuter de sujets spécifiques de celui-ci. Si votre projet s’étend sur plusieurs dépôts, vous pourriez utiliser des discussions d’organisation pour discuter des sujets qui ne sont pas spécifiques d’un dépôt unique dans votre organisation.

Les problèmes sont utiles pour discuter des détails spécifiques d’un projet, comme les rapports de bogues et les améliorations planifiées. Pour plus d’informations, consultez « [À propos des problèmes](/articles/about-issues) ». Les demandes de tirage vous permettent de commenter directement les changements proposés. Pour plus d’informations, consultez « [À propos des demandes de tirage](/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests) » et « [Commentaires dans une demande de tirage](/pull-requests/collaborating-with-pull-requests/reviewing-changes-in-pull-requests/commenting-on-a-pull-request) ».

## Suivi des recommandations en matière de contribution

Avant d’ouvrir une discussion dans un dépôt, vérifiez si le dépôt a des directives de contribution. Le fichier CONTRIBUTING comprend des informations sur la façon dont le chargé de maintenance du dépôt veut que vous contribuiez aux idées sur le projet.

Pour plus d’informations, consultez « [Configuration de votre projet pour des contributions saines](/communities/setting-up-your-project-for-healthy-contributions) ».
