## Cycle de publication de {% data variables.product.prodname_dotcom %}

Les produits et fonctionnalités {% data variables.product.prodname_dotcom %} peuvent passer par plusieurs phases de lancement.

| Phase | Description |
|-------|-------------|
| Alpha | Le produit ou la fonctionnalité est en cours de développement et ses exigences, de même que sa portée changent souvent. La fonctionnalité est disponible à des fins de démonstration et de test, mais peut ne pas être documentée. Les versions alpha ne sont pas nécessairement complètes en termes de fonctionnalités, aucun contrat de niveau de service (SLA) n’est fourni et il n’existe aucune obligation de support technique. <br><br> **Remarque** : un produit ou une fonctionnalité publié en tant que « Technology Preview » est considéré comme étant dans la phase de publication alpha. Les versions Technology Preview partagent les mêmes caractéristiques que les versions alpha décrites ci-dessus.|
| Bêta | Le produit ou la fonctionnalité est prêt à être plus largement distribué. Les versions bêta peuvent être publiques ou privées, sont documentées, mais n’ont pas de SLA ou d’obligations de support technique. |
| Disponibilité générale | Le produit ou la fonctionnalité est entièrement testé et ouvert publiquement à tous les utilisateurs. Les versions en disponibilité générale sont prêtes pour une utilisation en production, et les SLA associés et les obligations de support technique s’appliquent. |
