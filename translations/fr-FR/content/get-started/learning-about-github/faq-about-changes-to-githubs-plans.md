## Quels sont les changements de plans et de prix que GitHub a annoncé le 14 avril ?

GitHub a annoncé les changements de plans et de prix suivants :

- GitHub Free pour les organisations est immédiatement disponible et propose des dépôts privés pour un nombre illimité d’utilisateurs
- Toutes les organisations qui utilisaient précédemment Team for Open Source ont désormais GitHub Free
- GitHub Free pour les développeurs individuels comprend désormais un nombre illimité de collaborateurs
- Les organisations et les individus qui utilisent GitHub Free bénéficient de GitHub Community Support
- GitHub Pro comprend désormais 2 Go de stockage Packages et 10 Go de transfert de données
- Le prix mensuel de GitHub Pro est descendu à 4 USD
- Le prix mensuel de GitHub Team est descendu à 4 USD par utilisateur
- GitHub Team n’a plus besoin d’un minimum de cinq sièges
- GitHub Team comprend 3 000 minutes Actions par mois pour les dépôts privés

## Comment passer à un plan supérieur ?

Pour passer à un plan supérieur pour votre compte personnel, consultez « [Passer l’abonnement de votre compte personnel à un plan supérieur](/billing/managing-billing-for-your-github-account/upgrading-your-github-subscription#upgrading-your-personal-accounts-subscription) ». Pour passer à un plan supérieur pour votre organisation, consultez « [Passer l’abonnement de votre organisation à un plan supérieur](/billing/managing-billing-for-your-github-account/upgrading-your-github-subscription#upgrading-your-organizations-subscription) ».

## Comment passer à un plan inférieur ?

Pour passer à un plan inférieur pour votre compte personnel, consultez « [Passer l’abonnement de votre compte personnel à un plan inférieur](/billing/managing-billing-for-your-github-account/downgrading-your-github-subscription#downgrading-your-personal-accounts-subscription) ». Pour passer à un plan inférieur pour votre organisation, consultez « [Passer l’abonnement de votre organisation à un plan inférieur](/billing/managing-billing-for-your-github-account/downgrading-your-github-subscription#downgrading-your-organizations-subscription) ».

## Si je passe mon plan GitHub Team ou un plan hérité à GitHub Free, quelles sont les fonctionnalités que je vais perdre ?
- Réviseurs obligatoires dans les dépôts privés
- Branches protégées dans les dépôts privés
- Propriétaires de code dans les dépôts privés
- GitHub Pages dans les dépôts privés
- Wikis dans les dépôts privés
- Plusieurs destinataires pour les problèmes et les demandes de tirage dans les dépôts privés
- Plusieurs réviseurs de demande de tirage dans les dépôts privés
- Brouillon de demandes de tirage dans les dépôts privés
- Références liées automatiquement dans les dépôts privés
- Support standard

## Comment créer une organisation GitHub Free ou GitHub Team ?

Si vous avez un compte GitHub, créez une organisation dans notre [page de plan d’équipe](https://github.com/organizations/plan). Si vous n’avez pas de compte GitHub, vous pouvez [en créer un dans notre page d’accueil](https://github.com/), où vous êtes invité à créer une organisation pendant le processus d’inscription.

Vous pouvez voir les prix et la liste complète des fonctionnalités de chaque produit sur <{% data variables.product.pricing_url %}>.

## Combien de temps GitHub Free autorise un nombre de collaborateurs illimité ?

Indéfiniment ! Ce changement est permanent.

## J’ai déjà payé le plan annuel Pro ou Team, puis-je obtenir un remboursement pour la différence de prix ?

Les clients qui ont actuellement un abonnement annuel aux plans Pro et Team, et qui se sont inscrits avant le 14 avril recevront un remboursement calculé au prorata jusqu’à la fin de leur période de facturation annuelle. Par exemple, si cela fait deux mois que vous avez commencé votre abonnement annuel, vous recevez un remboursement de la différence de prix correspondant aux dix mois restants. Ces remboursements sont traités dans les 30 jours environ. Une fois votre remboursement traité, une notification est envoyée à l’e-mail de facturation.

Pour plus d’informations sur les différences entre chaque produit GitHub Free, consultez « [Produits {% data variables.product.prodname_dotcom %}](/articles/github-s-products) ».

## Comment obtenir plus de minutes Actions si j’ai besoin d’une quantité supérieure à celle allouée dans le plan GitHub Team ?

La plupart des petites équipes utilisent moins de 3 000 minutes Actions/mois, mais certaines équipes peuvent en utiliser plus. Si votre équipe a besoin de minutes supplémentaires pour les dépôts privés, vous pouvez augmenter votre limite de dépense pour GitHub Actions. Les minutes Actions et le stockage Packages sont toujours gratuits pour vos dépôts publics. Pour plus d’informations, consultez « [Gestion de votre limite de dépense pour GitHub Actions](/billing/managing-billing-for-github-actions/managing-your-spending-limit-for-github-actions) ».

## Comment les changements de la limite de sièges m’affectent-ils ?

Comme {% data variables.product.prodname_team %} n’a plus d’exigence minimale de sièges, si vous utilisez moins de cinq sièges dans votre organisation, vous pouvez supprimer les sièges payants pour réduire votre facture. Pour plus d’informations, consultez « [Passage de votre abonnement {% data variables.product.prodname_dotcom %} au niveau inférieur](/billing/managing-billing-for-your-github-account/downgrading-your-github-subscription#removing-paid-seats-from-your-organization) ».

## Est-ce qu’il y a des changements pour les équipes avec le plan GitHub Team for Open Source ?

Si votre compte d’organisation utilise actuellement le plan GitHub Team for Open Source, vous avez été automatiquement déplacé vers GitHub Free, qui comprend des dépôts privés avec un nombre illimité de collaborateurs, 2 000 minutes Actions pour les dépôts privés et GitHub Community Support. Les minutes Actions et le stockage Packages sont toujours gratuits pour vos dépôts publics.

## Qu’est-ce que GitHub Community Support ?

GitHub Community Support offre un support grâce à nos [discussions {% data variables.product.prodname_github_community %}](https://github.com/orgs/community/discussions) où vous pouvez consulter les solutions de la communauté GitHub, poser de nouvelles questions et partager des idées. GitHub Community Support rassemble des ingénieurs du support technique de GitHub Team, qui modèrent le {% data variables.product.prodname_github_community %} avec nos membres les plus actifs de la communauté. Si vous devez signaler des courriers indésirables, des abus ou que vous rencontrez des problèmes d’accès à votre compte, vous pouvez envoyer un message à notre équipe de support technique sur https://support.github.com/.

## Comment ce changement affecte-t-il les avantages éducatifs ?

Les étudiants et les enseignants continuent de bénéficier de tous les avantages qu’ils ont aujourd’hui. GitHub continue d’offrir GitHub Pro à tous les étudiants dans le cadre des avantages de son [pack de développement étudiant](https://education.github.com/pack). De plus, les enseignants continuent d’avoir accès à GitHub Team dans le cadre de la [boîte à outils de l’enseignant](https://education.github.com/toolbox).

## Comment cela m’affecte si je paye actuellement sur facture ?

Si vous payez sur facture, envoyez un e-mail à votre représentant commercial pour plus de détails. Si vous ne savez pas qui est votre représentant commercial, contactez votre responsable de facturation ou le propriétaire de votre compte.

## Quel est l’impact sur les utilisateurs et les organisations dans les régions où des sanctions du gouvernement américain sont appliquées ?

En raison des restrictions de la réglementation du commerce aux États-Unis, GitHub ne peut pas fournir des services de dépôt privé et des services payants aux comptes situés dans les régions américaines sanctionnées. GitHub a toutefois conservé l’accès aux services de dépôt public gratuits. Pour plus d’informations, consultez [GitHub et la réglementation du commerce](/free-pro-team@latest/github/site-policy/github-and-trade-controls).

GitHub continuera de plaider auprès des organismes de réglementation américains pour obtenir l’accès le plus vaste possible aux services de collaboration de code gratuit pour les développeurs des régions sanctionnées, y compris les dépôts privés gratuits. Nous croyons que l’offre de ces services gratuits soutient la politique étrangère des États-Unis visant à encourager la libre circulation de l’information et la liberté d’expression dans ces régions.
