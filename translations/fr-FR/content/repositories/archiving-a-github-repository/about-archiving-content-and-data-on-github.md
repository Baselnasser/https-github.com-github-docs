## Persistance des dépôts publics

{% data variables.product.product_name %} garde vos dépôts publics disponibles, sauf si vous les supprimez. Dans certains cas, nous pouvons rendre le contenu public indisponible, par exemple si :

- Nous recevons un [Avis de retrait DMCA](/free-pro-team@latest/github/site-policy/dmca-takedown-policy) pour le contenu d’un dépôt.
- Nous déterminons que le contenu d’un dépôt enfreint nos [Directives de la communauté](/free-pro-team@latest/github/site-policy/github-community-guidelines)ou nos [Conditions d’utilisation du service](/free-pro-team@latest/github/site-policy/github-terms-of-service).

Les universitaires et les chercheurs peuvent référencer ces informations dans des plans de gestion de données.

## À propos du {% data variables.product.prodname_archive %}

{% data reusables.repositories.about-github-archive-program %}

Le {% data variables.product.prodname_archive %} permet aux partenaires tiers d’archiver des dépôts publics avec l’API publique. Ces partenaires archivent différents types de données à différentes fréquences et rendent les données disponibles au public. Le {% data variables.product.prodname_archive %} protège également les données en continu en stockant plusieurs copies dans différents formats et emplacements. Par exemple, {% data variables.product.company_short %} stocke les dépôts dans le {% data variables.product.prodname_arctic_vault %}, une archive à très long terme destinée à durer au moins 1 000 ans. Pour plus d’informations, consultez [{% data variables.product.prodname_archive %}](https://archiveprogram.github.com/).

L’utilisation responsable des archives suppose le respect de la vie privée des utilisateurs. Pour plus d’informations, consultez « [Informations publiques sur GitHub](/free-pro-team@latest/github/site-policy/github-privacy-statement#public-information-on-github) ».

Vous pouvez désactiver le {% data variables.product.prodname_archive %} pour votre dépôt. Pour plus d’informations, consultez « [Activation ou désactivation du {% data variables.product.prodname_archive %} pour votre dépôt public](/get-started/privacy-on-github/opting-into-or-out-of-the-github-archive-program-for-your-public-repository) ».

## Ajout d’une licence open source pour augmenter l’archivabilité

Les bibliothèques et les chercheurs peuvent demander des protections juridiques pour créer des archives du contenu accessible au public. Si vous voulez que des tiers prennent en compte votre travail sur {% data variables.product.product_name %} pour l’archivage, vous pouvez ajouter une [licence open source](/articles/open-source-licensing) à vos projets. Une licence open source donne aux contributeurs des autorisations explicites pour copier et distribuer le contenu de vos dépôts.
