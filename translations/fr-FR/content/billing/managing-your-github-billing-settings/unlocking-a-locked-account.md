Vous pouvez déverrouiller votre compte et y accéder en mettant à jour le mode de paiement de votre organisation et en réactivant l’état payant. Nous ne vous demandons pas de payer le temps écoulé en mode verrouillé.

Vous pouvez faire passer votre organisation à {% data variables.product.prodname_free_team %} pour continuer avec les mêmes fonctionnalités avancées dans les dépôts publics. Pour plus d’informations, consultez « [Passage de votre abonnement {% data variables.product.product_name %} au niveau inférieur](/billing/managing-billing-for-your-github-account/downgrading-your-github-subscription) ».

## Déverrouillage des fonctionnalités d’une organisation en raison d’un paiement refusé

Si les fonctionnalités avancées de votre organisation sont verrouillées en raison d’un paiement refusé, vous devez mettre à jour vos informations de facturation pour autoriser un nouveau prélèvement. Pour plus d’informations, consultez « [Ajout ou modification d’un mode de paiement](/articles/adding-or-editing-a-payment-method) ».

Si les nouvelles informations de facturation sont approuvées, nous vous facturons immédiatement le produit payant que vous avez choisi. L’organisation se déverrouille automatiquement quand un paiement réussi a été effectué.
