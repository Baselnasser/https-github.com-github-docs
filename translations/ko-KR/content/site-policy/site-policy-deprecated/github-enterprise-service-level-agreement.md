이러한 조건은 2021년 1월 4일 이전에 제품 라이선스를 받은 고객에게 적용됩니다. 해당 날짜 이후에 GitHub 제품을 구매한 고객은 https://www.github.com/enterprise-legal 의 현재 약관으로 안내됩니다.

**짧은 버전:** GitHub는 해당 GitHub 서비스에 대해 99.9%의 분기별 작동 시간(“**서비스 수준**” 또는 “**SLA**”) 약정을 보장합니다. GitHub가 SLA를 충족하지 못하는 경우, 고객은 고객 계정으로 서비스 크레딧(“**서비스 크레딧**”)을 받을 권한이 있습니다.

각 서비스 기능(“**서비스 기능**”)에 대한 정의를 확인하고 과거 및 현재 작동 시간을 살펴보려면 [GitHub 상태 페이지](https://www.githubstatus.com/)를 방문하십시오. 이 SLA에 정의되지 않고 사용되는 특정 용어는 고객의 해당 계약에서와 동일한 의미를 갖습니다.

## 작동 시간 보장

“**작동 시간**”이란 주어진 분기 내에 해당 GitHub 서비스의 총 가능 가용 시간(분)의 백분율입니다. GitHub는 해당 GitHub 서비스에 대해 99.9% 이상의 작동 시간 유지를 약정합니다. 해당 GitHub 서비스에 포함될 수 있는 각 서비스 기능에 대한 작동 시간 계산은 아래(“**작동 시간 계산**”)에 설명되어 있습니다. GitHub가 SLA를 충족하지 못하는 경우, 고객은 아래 계산(“**서비스 크레딧 계산**”)에 따른 서비스 크레딧을 받을 자격이 생깁니다. 가동 중지 시간은 모든 고객에게 동시에 또는 같은 방식으로 영향을 주지는 않습니다.

| **서비스 기능** | **작동 시간 계산** | **정의** | **서비스 크레딧 계산** |
|---|---|---|---|
| **문제**,<br>**끌어오기 및 요청**,<br>**Git 및 작동**,<br>**API 및 요청(서비스 기능에만 해당)** ,<br>**웹후크**,<br>**페이지** | (한 분기의 총 시간(분) - 가동 중지 시간)/한 분기의 총 시간(분) | “**가동 중지 시간**”은 (a) 어떤 서비스 기능에 대해 주어진 1분 내에 오류율이 오 퍼센트(5%)를 초과하는 경우 또는 (b) GitHub의 내외부 모니터링 시스템을 조합하여 판단할 때 서비스 가용성이 확보되지 않은 경우 중 하나가 발생한 시간을 말합니다. | 서비스 크레딧의 청구는 다음 계산 중 하나(둘 다는 아님)를 기반으로 할 수 있습니다. <ul><li>해당 서비스 기능의 작동 시간이 99.9%보다 작거나 같았지만 99.0%보다 컸던 분기의 서비스 기능에 대해 고객이 지불한 금액의 10%. <BR><BR>또는 <BR><BR></li><li>해당 서비스 기능의 작동 시간이 99.0% 미만이었던 분기의 서비스 기능에 대해 고객이 지불한 금액의 25%.</li></ul> | |
| **actions** | (트리거된 총 실행 수 – 실행 실패 수)/(트리거된 총 실행 수) x 100 | “**트리거된 실행 수**”란 한 분기에 고객이 트리거한 모든 작업의 총수입니다. <br><br> “**실행 실패 수**”란 분기에 트리거된 총 실행 수 이내에서 작동에 실패한 총 실행 수입니다.  실행 실패는 트리거가 정상 작동된 후 오(5) 분 후에도 작업 기록 로그가 출력을 캡처하지 않을 때 발생합니다. | 위와 동일 |
| **패키지** | 전송 작동 시간 = 작업과 동일 <br> <br> 스토리지 작동 시간 = 100% - 평균 오류 비율* <br> <br> *작동 시간 계산에서 공개 사용량 및 총 스토리지 트랜잭션 또는 실패한 스토리지 트랜잭션에 계산되지 않은 스토리지 트랜잭션은 제외됩니다(사전 인증 실패, 인증 실패, 지정된 할당량을 초과한 스토리지 계정에 대해 시도된 트랜잭션 포함). | “**오류 비율**”은 설정된 시간 간격(현재 1시간으로 설정됨)에 전체 스토리지 트랜잭션으로 나눈 실패한 스토리지 트랜잭션의 총수입니다. 지정된 한 시간 간격에 총 스토리지 트랜잭션이 0인 경우 해당 간격에 대한 오류 비율은 0%입니다. <br><br> “**평균 오류 비율**”이란 한 분기 내의 각 시간에 대한 오류 비율의 합을 한 분기 내의 총 시간으로 나눈 값입니다. | 위와 동일 |

## 제외
다음으로 인한 서비스 기능 실패는 작동 시간 계산에서 제외됩니다. (i) 고객의 작위, 부작위 또는 계약 위반을 포함한 해당 GitHub 서비스의 오용, (ii) 고객의 인터넷 연결 실패, (iii) 불가항력 사건을 비롯해 GitHub의 합리적인 통제 밖에 있는 요소, 또는 (iv) 고객의 장비, 서비스 또는 기타 기술.

## 서비스 크레딧 사용
GitHub가 이 SLA를 충족하지 않는 경우 고객은 한 분기 종료 후 30일 이내에 GitHub에 서면으로 요청한 경우에만 서비스 크레딧을 사용할 수 있습니다. 서비스 크레딧 사용 및 GitHub Enterprise Cloud 사용자 지정 월간 또는 분기별 보고서에 대한 서면 요청을 [GitHub 지원](https://support.github.com/contact?tags=docs-policy)으로 보내야 합니다.

서비스 크레딧은 고객 계정으로의 환불 또는 크레딧 형식이 될 수 있으며, 현금으로 교환할 수 없고, 분기당 최대 유료 서비스 구십(90) 일로 제한되며, 고객이 미결제 청구서를 지불해야 하며, GitHub와 고객의 계약 종료 시에 만료됩니다. 서비스 크레딧은 GitHub가 이 SLA의 의무를 충족하지 못하는 일체의 불이행에 대한 유일하고 배타적인 구제 수단입니다.
