본 계약은 아래에 자세히 정의된 바와 같이 다음과 같은 GitHub 제품(총칭하여 **“제품”** )에 적용됩니다.

-   GitHub Enterprise(추가 기능 소프트웨어가 포함될 수 있는 GitHub Enterprise Server 및 GitHub Enterprise Cloud로 구성됨)

-   베타 미리 보기

-   관련된 모든 지원

-   관련된 모든 전문 서비스

본 계약에는 다음 섹션과 별첨이 포함되며, 각각은 참조에 의해 본 계약에 포함됩니다.

* 섹션 1: 일반 약관

* 섹션 2: GitHub Enterprise Server 사용 조건

* 섹션 3: GitHub Enterprise Cloud 서비스 약관

* 별첨 A: 정의.

고객이 허가된 GitHub 재판매인에게 제품을 구입한 경우 본 계약의 섹션 1.1 (기간 및 해지), 1.2 (지급), 섹션 2.3 (배송), 섹션 2.4 (인증)보다 고객이 GitHub 재판매인과 합의했던 조건이 우선합니다.

## 섹션 1: 일반 약관

본 섹션 1은 고객의 구매 및 제품 사용에 적용되는 약관을 명시합니다. 본 계약의 본문에 사용되었지만 정의되지는 않은 특정 용어의 경우 별첨 A에 정의됩니다.

### 1.1 기간 및 해지

#### 1.1.1 기간.

본 계약은 발효일에 시작되어 이 섹션 1.1에 따라 당사자가 해지할 때까지 그 효력을 계속 유지합니다.

#### 1.1.2 편의에 의한 해지. 계정 취소

각 당사자는 해당 구독 기간이 끝나기 전에 적어도 30일 기한의 사전 서면 통지를 통해 특별한 사유 없이 주문서 양식 또는 본 계약을 해지할 수 있습니다. 고객이 주문서 양식 또는 계약을 해지하기로 선택한 경우, 화면 상단의 전역 탐색 모음에서 Settings(설정)로 이동하여 GitHub 계정을 적절하게 취소할 책임은 고객에게 있습니다. GitHub는 이메일 또는 전화 요청에 대한 응답으로 계정을 취소할 수 없습니다.

#### 1.1.3 중대한 위반에 따른 해지.

각 당사자는 계약 상대방이 본 계약에 따른 중대한 의무를 위반하고 통지를 받은 날로부터 30일 이내에 위반을 시정하지 않으면 통지 즉시 본 계약을 해지할 수 있습니다. 또한 각 당사자는 서면 통지를 통해 보정기간(cure period)을 제공하지 않고 즉시 본 계약을 다음과 같이 해지할 권리를 보유합니다. 즉, 고객이 GibHub의 지식재산과 관련하여 본 계약의 조건을 위반하는 경우(라이선스 허여 또는 라이선스 제한 사항과 관련한 고객의 규정 미준수 포함), 또는 고객의 계정이 구십(90) 일 넘게 중지된 경우에는 GitHub에 의해, 또는 상대방이 섹션 1.4의 기밀 유지 의무를 위반한 경우 당사자 중 하나에 의해 해지할 권리입니다.

#### 1.1.4 해지의 효력 및 존속.

본 계약이 해지되면 고객은 추가 주문서 양식을 실행할 수 없습니다. 다만 본 계약은 진행 중인 주문서 양식의 남은 기간 동안 유효합니다. 주문서 양식의 구독이 해지되거나 만료되는 경우 동 양식과 관련하여 (i) 소프트웨어 및/또는 서비스에 대한 구독 기간은 즉시 종료되고, (ii) 주문서 양식의 모든 구독 라이선스는 자동으로 해지되며 고객은 더 이상 제품을 사용할 권리가 없으며, (iii) 구독 해지 전에 지불해야 할 요금이 남아 있다면 고객은 해당 요금을 즉시 지불해야 하며, (iv) 고객은 고객이 소지하거나 제어하고 있는 모든 소프트웨어 카피를 파기하고 고객이 그렇게 했음을 GitHub에 서면으로 확인해야 하며, (v) 각 당사자는 서비스가 허용하는 범위 내에서 상대방에게 속한 모든 비밀 정보를 즉시 반환(또는 상대방이 요청할 경우 파기)합니다. 상기 내용에도 불구하고 고객은 고객의 데이터를 마이그레이션하는 소프트웨어에 액세스할 수 있고, 본 계약 또는 주문서 양식의 해지 또는 만료 이후 최대 구십(90) 일까지 고객의 리포지토리에 있는 데이터 마이그레이션을 요청할 수 있습니다. 하지만 고객은 해당 시간 동안 생산을 근거로 소프트웨어나 서비스를 사용해서는 안 됩니다. 성격상 합리적으로 효력이 유지되어야 하는 모든 조항은 본 계약 또는 주문서 양식의 해지 또는 만료 후에도 효력이 유지됩니다.

### 1.2 지급.

#### 1.2.1 요금.

고객은 어떠한 종류의 공제 또는 상계 없이 미국 달러로 요금을 전액 선불하는 데 동의합니다. 고객은 GitHub 청구일로부터 30일 이내에 요금을 지불해야 합니다. 본 계약에 따라 지불할 금액은 섹션 1.5.1 및 1.6.2에 명시된 경우를 제외하면 환불되지 않습니다. 고객이 요금을 제때 지불하지 않을 경우, GitHub는 법률 또는 형평상 그 밖의 모든 조치를 취하는 것 외에도 (i) 연체 미지불금에 대해 월 1.0% 이율 또는 법정 허용 최고 이율 중 낮은 이율을 기준으로 이자를 청구하고 모든 회수 비용을 청구할 권리가 있으며 (ii) 해당 주문서 양식 또는 SOW를 해지할 권리가 있습니다. 고객은 본 계약과 관련하여 부과되거나 납부해야 할 모든 세금, 요금, 관세 및 정부 과세액(GitHub의 순수입에 근거하여 부과되는 제세공과금은 제외)에 대해 전적인 책임을 집니다.

#### 1.2.2 추가 구독 라이선스 구입.

고객은 GitHub의 웹사이트 또는 해당 영업 팀을 통해 요청을 제출하여 본 계약에 따라 추가 구독 라이선스를 얻을 수 있습니다. 추가 구독 라이선스를 구입하는 경우, 고객은 해당 구독 약정 기간의 잔액에 대해 일할 계산된 금액으로서 당시 적용 가능한 요금을 지불해야 합니다. 다른 구독 약정 기간 동안 고객의 구독 라이선스를 갱신할 때 GitHub는 주문서 양식에 별도의 요건을 명시하지 않는 한, 모든 구독 라이선스에 대한 요금을 매년 한 번 청구합니다.

#### 1.2.3 추가 기능 소프트웨어.

추가 기능 소프트웨어는 사용자당 기준으로 라이선스가 부여됩니다. 의심의 여지를 피하기 위하여 주문서 양식에 달리 명시되지 않는 한, 고객이 추가 기능 소프트웨어에 대해 지정된 시간에 보유한 구독 라이선스 수는 고객이 본 계약에 따라 제품에 대해 보유한 구독 라이선스 수와 같아야 합니다. 예를 들어 고객이 Advanced Security에 대한 구독을 구매하고자 하고 해당 제품의 사용자 100명에 대한 구독 라이선스를 이미 보유하고 있는 경우 Advanced Security 사용자 100명에 대한 구독 라이선스를 구매해야 합니다.

### 1.3 전문 서비스.

전문 서비스에 대한 고객의 요청에 따라 GitHub는 이러한 전문 서비스를 자세히 설명한 SOW를 제공합니다. GitHub는 각 SOW에 설명된 전문 서비스를 수행합니다. GitHub는 전문 서비스를 수행하는 방식과 수단을 관리하며, 배정된 인원을 결정할 권리가 있습니다. GitHub는 전문 서비스를 수행하기 위해 제3자를 사용할 수 있습니다. 단, GitHub는 제3자의 작위 및 부작위에 대해 여전히 책임이 있습니다. 고객은 소프트웨어, 도구, 사양, 아이디어, 개념, 발명, 프로세스, 기술 및 노하우를 포함해 전문 서비스 수행과 관련하여 사용되거나 개발된 모든 것에 대한 권리, 소유권 및 이권이 모두 GitHub에 있음을 인정하고 이에 동의합니다. GitHub는 전문 서비스를 수행하는 동안 고객에게 무엇이든 제공하는 한, 본 계약 기간 동안에만 해당 제공물을 사용할 수 있는 권리로서 비독점적이고 양도 불가능하며 사용 기한이 있는 전 세계적 라이선스를 로열티 없이 오로지 고객의 소프트웨어 또는 서비스 사용과 관련하여 고객에게 허여합니다.

### 1.4 기밀 유지.

본 계약에서 허용하는 경우를 제외하면 어떤 당사자도 상대 당사자의 비밀 정보를 사용하지 않습니다. 각 당사자는 유사한 성격의 자체 정보에 대해 적용하는 것과 똑같은 수준의 주의를 기울이면서도 어떤 경우든 간에 최소한 합당한 수준의 주의를 기울여 상대 당사자의 비밀 정보를 비밀로 유지하고 보호하는 데 동의합니다. 각 당사자는 상대방의 기밀 정보가 무단 공개되는 것을 방지하기 위해 합당한 모든 예방 조치를 취하는 데 동의합니다. 여기에는 해당 기밀 정보가 (i) 비밀 정보를 알아야 할 필요가 있고, (ii) 이 섹션 1.4를 준수하기 위해 충분히 적절한 계약을 체결한 당사자이며, (iii) 이 섹션 1.4에 명시된 사용 및 공개에 관한 제한 사항을 통지받는 대리인에게만 공개되도록 하는 것이 포함되나 이에 국한되지 않습니다. 각 당사자는 해당 대리인의 모든 작위 및 부작위에 대해 책임이 있습니다. 앞서 언급한 제반 의무는 법원, 행정 기관 또는 그 외 정부 기관의 명령이나 요구 사항에 따라 어느 한 당사자가 상대 당사자의 비밀 정보를 공개하는 것을 제한하지 않습니다. 단, 그러한 정보 공개를 해야 하는 당사자는 그러한 명령 또는 요구 사항에 대해 이의를 제기할 수 있도록 법률상 금지된 경우가 아니라면 상대 당사자에게 합당한 통지를 전달해야 합니다. 이 섹션 1.4에 명시된 제한 사항은 본 계약의 해지 또는 만료 후에도 유효합니다.

### 1.5 청구에 대한 방어.

양 당사자는 이 섹션 1.5에 명시된 요건 및 범위 내에서 제3자 청구로부터 서로를 방어하며 불리한 최종 판결 또는 승인된 합의 결과에 따른 비용을 지불합니다. 단, 이 경우 방어 당사자는 해당 청구에 대해 즉시 서면 통지를 받고 방어 및 모든 합의를 통제할 권리가 있어야 합니다. 방어를 받는 당사자는 방어하는 당사자에게 요청된 모든 지원, 정보 및 권한을 제공해야 합니다. 방어 당사자는 지원을 제공하는 과정에서 발생하는 합당한 현금 지출 비용을 상대 당사자에게 배상하며, 불합리하게 배상을 보류 또는 지연하지 않도록 상대 당사자의 사전 서면 동의 없이 제3자 청구와 관련하여 합의를 하거나 수락하지 않습니다. 이 섹션 1.5는 그러한 청구에 대한 양 당사자의 유일한 구제 수단과 전반적인 책임을 설명합니다.

#### 1.5.1 GitHub.

GitHub는 소프트웨어 또는 서비스에 대한 고객의 승인된 이용이 저작권, 특허 또는 상표를 침해하거나 비계열사인 제3자의 영업 비밀을 악용한다는 혐의를 주장하는 비계열사인 제3자가 제기한 모든 청구에 대해 고객을 방어합니다. GitHub가 상업적으로 합당한 조건 하에서 그러한 청구를 해결할 수 없는 경우, 다음 중 한 가지 조치를 선택하여 취할 수 있습니다. 즉, (i) 소프트웨어 또는 서비스를 수정, 수리 또는 교체(해당되는 경우)하거나 (ii) 고객의 구독을 해지하고 사용하지 않은 선불 구독료를 환불합니다. GitHub는 다음과 같은 원인으로 인해 발생하는 그러한 청구에 대해 이 섹션 1.5.1에 따른 의무를 지지 않습니다. (a) 본 계약(문서 포함)에서 명시적으로 승인한 범위를 벗어나는 소프트웨어 또는 서비스의 변경, 또는 본 계약(설명서 포함)에서 명시적으로 승인한 범위를 벗어나 장비, 장치, 소프트웨어, 시스템 또는 데이터와 함께 소프트웨어 또는 서비스를 조합, 운영 또는 사용, (b) 고객이 소프트웨어 또는 서비스 사용 중지에 관한 통지를 받은 후 이를 따르지 않는 경우, (c) 섹션 1.5.2에 따른 고객의 의무, (d) GitHub에서 무료로 제공하는 제품 또는 서비스(소프트웨어 또는 서비스 이용 포함) 또는 (e) 베타 미리 보기의 액세스 또는 사용. 이 섹션 1.5.1에 따른 GitHub의 의무를 이행하기 위해 소프트웨어 및 서비스는 동 조항에서 GitHub에 의해 통합된 오픈 소스 구성 요소를 포함합니다.

#### 1.5.2 고객.

고객은 계열사가 아닌 제3자가 다음과 같은 항목으로 인해 제기하는 모든 청구에 대해 GitHub를 방어합니다. 즉, (i) 소프트웨어 또는 서비스에 업로드하는 고객 콘텐츠, (ii) 고객의 본 계약 위반(고객의 기밀 유지 위반, GitHub의 허용 가능한 사용 약관의 위반 포함), (iii) 고객의 소프트웨어에 대한 수정, 또는 (iv) 고객이 소프트웨어 또는 서비스와 함께 조합, 운영 또는 사용하는 제3자 브랜드 장비, 장치, 소프트웨어, 시스템 또는 데이터 등입니다.

### 1.6 진술 및 보증, 보증 부인, 책임의 제한.

#### 1.6.1 일반 보증.

각 당사자는 본 계약을 체결할 법적 권능과 권한이 있으며 본 계약의 계약조건에 따른 구속력을 그러한 당사자에게 행사하는 데 필요한 모든 권한을 가진 당사자의 직원 또는 대리인이 본 계약과 각 주문서 양식 및 SOW 약정을 체결함을 계약 상대방에게 진술하고 보증합니다.

#### 1.6.2 제한적 보증.

**(i) 소프트웨어.** GitHub는 (1) 수정되지 않은 소프트웨어가 다운로드를 위해 고객에게 제공된 시점에 멜웨어, 바이러스 또는 웜(고객의 소프트웨어, 하드웨어, 컴퓨터 시스템 또는 네트워크를 중단, 무력화하거나 피해를 입히기 위해 특별히 설계된 컴퓨터 코드 또는 기타 기술이라고도 알려짐)을 포함하거나 전송하지 않으며, (2) 최초 다운로드를 위해 제공된 날짜에서 구십(90) 일 동안 수정되지 않은 소프트웨어가 실질적으로 해당 설명서와 일치함을 보증합니다. GitHub는 고객의 소프트웨어 사용이 중단되지 않을 것임 또는 소프트웨어 작동에 오류가 없을 것임을 보증하지 않습니다. 본 섹션 1.6.2(i)의 보증은 고객이 섹션 2 및 설명서에 명시적으로 허용하지 않은 일체의 방식으로 소프트웨어를 수정 또는 사용하는 경우 적용되지 않습니다. GitHub의 유일한 의무 및 본 보증의 위반에 대한 고객의 유일한 구제책은 GitHub의 선택과 비용 부담으로 다음 중 하나를 하는 것입니다. 즉, (a) 소프트웨어를 수리하거나, (b) 소프트웨어를 교체하거나 또는 (c) 결함 소프트웨어에 관한 본 계약을 해지하고 당시 구독 기간 중 결함 소프트웨어에 대한 미사용 선불 요금을 환불하는 것 중 하나입니다.

**(ii) 전문 서비스.** GitHub는 적절한 자격을 갖춘 직원이 본 계약에 따른 모든 전문 서비스를 전문적이면서도 숙련된 방식으로 수행함을 고객에게 보증합니다. 이러한 보증의 위반에 대한 GitHub의 유일한 의무이자 고객의 유일한 구제책은 GitHub의 선택과 비용 부담을 조건으로 (1) 이 보증을 충족하지 못하는 모든 전문 서비스를 즉시 다시 수행하거나 (2) 해당 위반을 시정할 수 없다면 SOW의 약정을 해지하고 사용하지 않은 선불 요금을 환불하는 것입니다.

#### 1.6.3 보증 부인.

**(i) 일반 조항.** 명시적으로 본 계약에 명시된 경우를 제외하고, GitHub는 어떤 종류의 다른 보증 및 진술을 하지 않으며 본 계약에 따라 상업성, 특정 목적에의 적합성, 비침해성 또는 거래 과정 또는 상관습에서 비롯된 일체의 보증 또는 조건을 포함하되 이에 국한되지 않는 다른 모든 명시적, 묵시적 또는 법적 보증을 명시적으로 부인합니다. 구두 또는 서면에 관계없이 GitHub 또는 다른 곳에서 제공된 조언 또는 정보는 본 계약에 명시적으로 진술되지 않은 일체의 보증 또는 조건을 구성하지 않습니다.

**(ii) 서비스.** GitHub는 어떠한 종류의 보증 없이 서비스를 **"있는 그대로"** 그리고 **"사용할 수 있는 상태로"** 제공합니다. 이러한 요건을 제한하지 않고 GitHub는 상업성, 특정 목적에 대한 적합성, 소유권, 보안, 정확성 및 비침해성에 대한 보증을 포함하되 이에 국한되지 않는 서비스에 관한 명시적, 묵시적 또는 법적인 모든 보증을 명시적으로 부인합니다. GitHub는 서비스가 고객의 요구 사항을 충족한다는 점, 서비스가 중단되지 않고 제때 제공되며 안전하거나 오류가 없을 것이라는 점, 서비스를 통해 제공되는 정보가 정확하거나 신뢰할 수 있거나 정확하다는 점, 모든 결함이나 오류는 수정될 것이라는 점, 서비스가 특정 시간 또는 장소에서 제공될 것이라는 점, 또는 서비스에 바이러스 또는 그 외 유해한 구성 요소가 없다는 점 등을 보증하지 않습니다. GitHub는 고객이 서비스에서 얻은 파일, 정보, 콘텐츠 또는 기타 자료를 다운로드하거나 사용함으로써 발생하는 손실 위험에 대해 책임을 지지 않습니다.

**(iii) 베타 미리 보기.** 고객은 본인의 선택에 따라 단독 재량으로 베타 미리 보기를 사용할 수 있습니다. 베타 미리 보기는 지원되지 않을 수 있으며 통지 없이 언제든지 변경할 수 있습니다. 베타 미리 보기는 서비스만큼 안정적이거나 사용 가능하지 않을 수도 있습니다. 베타 미리 보기는 지금까지 그리고 현재에도 서비스에 적용되고 있는 동일한 보안 조치 및 감사가 적용되지 않습니다. GitHub는 베타 미리 보기로 인해 또는 이와 관련하여 발생하는 책임을 지지 않습니다. **고객은 본인의 위험 부담 하에 베타 미리 보기를 사용합니다.**

#### 1.6.4 책임의 제한.

**(i)   간접적 손해. 관련 법률이 최대한 허용하는 범위 내에서 각 당사자는 어떠한 경우에도 계약 상대방 또는 제3자에게 간접적, 특수적, 우발적, 징벌적 또는 결과적 손해(이익, 수익 또는 데이터의 손실을 포함)에 대한 책임을 지지 않으며, 그러한 책임이 계약, 보증, 불법 행위(과실 포함), 무과실 책임 등을 근거로 한 모든 청구에서 발생하거나 그렇지 않으면 한 당사자가 그러한 손해의 발생 가능성에 관한 알림을 받았는지 여부에 관계없이 본 계약으로 인해 또는 본 계약과 관련하여 발생하는 대체 제품을 획득하는 데 드는 비용에 대해서도 책임을 지지 않습니다.**

**(ii) 총 책임의 제한. 관련 법률이 최대한 허용하는 범위 내에서 모든 소송 원인 및 모든 책임 이론에 근거하여 본 계약에 따라 각 당사자가 부담해야 할 총 누적 책임은 어떤 경우든 간에 그러한 책임을 야기하는 청구가 발생하기 직전 12개월 동안 고객이 GitHub에 실제로 지불한 요금을 초과하지 않습니다. 무료로 제공되는 제품 및 서비스(제품 사용을 포함)의 경우, GitHub의 책임 한도액은 최대 미화 5천 달러의 직접 손해로 제한됩니다. 베타 미리 보기의 경우, GitHub의 책임 한도액은 최대 미화 500달러의 직접 손해로 제한됩니다.**

**(iii) 예외. 이 섹션 1.6.4에 명시된 제외 및 제한 요건은 (1) 당사자가 섹션 1.4의 기밀 유지 의무를 위반한 이유로 발생하는 책임(상기의 제한 및 제외 요건이 계속 적용되는 콘텐츠(GitHub 콘텐츠는 제외)와 관련된 모든 책임은 예외로 함)에는 적용되지 않으며, (2) 섹션 1.5에 명시된 당사자의 방어 의무, 또는 (3) 섹션 2.1 및 2.2의 라이선스 허여 또는 라이선스 제한 사항에 대한 고객의 미준수에 의해 발생하는 책임에는 적용되지 않습니다.**

### 1.7 구독 라이선스.

구독 라이선스는 사용자별로 허여되며 여러 사용자가 동일한 구독 라이선스를 사용할 수는 없습니다. 라이선스 재할당의 원인이 (i) 하드웨어의 영구적인 오류나 손상, (ii) 사용자의 고용 또는 계약 해지, 또는 (iii) 사용자의 부재를 다루기 위한 구독 라이선스의 임시 재할당에서 비롯된 것이 아닌 한, 고객은 동일한 구독 라이선스를 마지막으로 재할당한 후 90일이 경과한 후에만 새 사용자에게 구독 라이선스를 재할당할 수 있습니다. 고객이 한 사용자의 구독 라이선스를 다른 사용자에게 재할당하는 경우, 고객은 구독 라이선스 및 고객의 조직에 대한 이전 사용자의 액세스를 차단해야 합니다.

### 1.8 계열사.

고객이 소프트웨어 및 서비스의 액세스 및 사용에 대해 여전히 전적인 책임을 지는 한, 고객의 계열사는 본 계약에 따라 소프트웨어 및 서비스를 사용할 권한이 있습니다.

### 1.9 지원.

GitHub는 소프트웨어 및 서비스에 대해 다음과 같은 지원을 제공합니다.

#### 1.9.1 표준 지원.

GitHub는 주말 및 미국 국경일을 제외하면 하루 24시간, 주 5일 동안 추가 비용 부담 없이 소프트웨어 및 서비스에 대한 표준 기술 지원을 제공합니다. 표준 지원은 GitHub 지원을 이용한 웹 기반 티켓팅을 통해서만 제공되며, 지원 요청은 GitHub의 지원 팀과 상호 작용할 수 있는 사용자로부터 시작되어야 합니다.

#### 1.9.2 향상된 지원 제품.

GitHub는 [지원 약관](/enterprise/admin/enterprise-support/about-github-premium-support-for-github-enterprise) 및 주문서 양식 또는 SOW에 지정된 지원 수준, 요금 및 구독 기간에 따라 소프트웨어 및 서비스에 대한 향상된 지원 제품(프리미엄, 프리미엄 플러스 및 엔지니어링 직접 지원 제공 포함)을 제공할 수 있습니다.

#### 1.9.3 예외.

GitHub는 고객이 GitHub에 알린 소프트웨어의 모든 중대하고 재현 가능한 오류를 수정하기 위해 합리적인 노력을 기울입니다. 그러나 GitHub는 다음과 같은 경우 지원을 제공할 책임을 지지 않습니다. 즉, (i) (GitHub 이외의) 누군가가 소프트웨어를 수정하는 경우, (ii) 고객이 소프트웨어 또는 성능에 부정적인 영향을 주는 방식으로 운영 체제 또는 환경을 변경한 경우, (iii) 고객이 본 계약 또는 설명서에 따라 승인된 것 이외의 방식으로 소프트웨어를 사용한 경우, 또는 (iv) 소프트웨어에 대한 고객의 과실 또는 오용이 있는 경우입니다.

### 1.10 지적 재산권.

양 당사자 가운데 GitHub는 제품에 대한 모든 지적 재산권을 포함하여 모든 권리, 소유권 및 이권을 소유합니다. GitHub는 본 계약에 따라 고객에게 명시적으로 부여되지 않은 권리로서 제품에 귀속된 모든 권리를 보유합니다. GitHub는 고객이 제공하거나 포럼에 게시할 수 있는 모든 피드백, 의견 또는 제안을 고객에 대한 어떠한 의무 없이 사용 및 수정하거나 자사 제품에 통합할 수 있습니다.

### 1.11 피드백.

고객은 제품에 관한 피드백을 GitHub에 제공할 수 있습니다. 피드백은 자발적이며 고객 비밀 정보에 속하지 않습니다(심지어 그와 같이 지정된 경우에도). GitHub는 (i) GitHub의 기존 및 향후 제품군의 운영, 기능 및 사용을 개선하고 그러한 제품군을 상용화하기 위해 그리고 (ii) 제품의 품질에 관하여 집계된 통계를 공개하기 위해 그러한 피드백을 충분히 행사하고 활용할 수 있습니다. 단, 그러한 공개 자료의 데이터는 고객과 그 직원 또는 고객의 독점 소프트웨어 코드를 구체적으로 식별하는 데 사용되지 않습니다.

### 1.12 법률 및 규정 준수.

고객은 제품 사용에 있어서 데이터 보호 및 고용 법률과 규정을 포함하지만 이에 국한되지 않는 모든 관련 법률 및 규정을 준수합니다.

### 1.13 일반 조항.

#### 1.13.1 준거법, 재판지.

본 계약의 어떤 조항이 관할 법원에서 불법, 무효 또는 이행 불가능한 것으로 판정되는 경우, 양 당사자는 해당 조항의 효력을 최대한 발생할 수 있도록 본 계약을 수정하거나 개정합니다. 이러한 방식으로 수정하거나 개정할 수 없는 모든 조항은 삭제된 것으로 간주되며 본 계약의 나머지 조항은 계속해서 완전한 효력을 유지합니다.


## 섹션 2: GitHub Enterprise Server 사용 조건

본 섹션 2에는 고객의 소프트웨어 사용에 적용되는 용어가 자세히 설명됩니다.

### 2.1 라이선스 허여.

GitHub는 고객에게 고객의 내부 비즈니스 목적을 위해 해당 구독 기간 동안 설명서에 따라 고객의 주문서 양식에 명시된 구독 라이선스 개수에 대해서만 소프트웨어를 설치하고 사용할 수 있는 비배타적이며, 이전 불가하며, 전 세계적이며, 로열티가 없는, 기간 제한적인 라이선스를 허여합니다. 소프트웨어에는 제3자가 GitHub에 라이선스를 허여한 구성 요소가 포함되는데, 여기에는 GitHub가 해당 구성 요소에 대한 소스 코드를 제공하도록 요구하는 라이선스를 가진 소프트웨어가 포함됩니다. 해당 구성 요소에 대한 소스 코드는 요청 시 제공됩니다. 상기 내용을 제한하지 않고 본 라이선스는 고객이 Microsoft SQL Server Standard Edition Linux 파일용 컨테이너 이미지(“**SQL Server Images**”)를 다운로드하고 실행하도록 허가하며, 이는 문서화된 대로 소프트웨어와 함께만 사용할 수 있습니다. 고객이 SQL Server Images를 사용할 권한은 고객이 소프트웨어를 사용할 권한이 더 이상 없을 때 종료되며, 고객은 SQL Server Images의 사용 권한이 종료되면 이를 삭제해야 합니다. Microsoft Corporation은 언제든지 SQL Server Images를 사용 중지할 수 있습니다.

### 2.2 라이선스 제한.

법률에 의해 또는 해당 제3자 라이선스에 의해 명시적으로 허가된 경우를 제외하고, 고객 및 그 계열사가 다음 행위를 하거나, 제3자가 하도록 허용해서는 안 됩니다. 즉, (i) 소프트웨어의 라이선스 재허여, 판매, 대여, 리스, 이전, 양도 또는 재배포, (ii) 제3자의 이익을 위한 소프트웨어 호스팅, (iii) 본 계약 섹션 2에 명시적으로 허용한 경우를 제외하고 제3자가 소프트웨어에 액세스하도록 공개 또는 허가, (iv) 라이선스 키의 해킹이나 수정 또는 라이선스 등록 과정의 회피 또는 변경, (v) 고객 수정을 제외한 소프트웨어 파생 저작물 수정 또는 생성, 또는 소프트웨어의 다른 소프트웨어와의 병합, (vi) 소프트웨어의 분해, 디컴파일, 코드 이분 우회 또는 소프트웨어의 전체 또는 부분의 소스 코드 도출 시도, (vii) 소프트웨어 또는 설명서에 포함된 모든 독점 권리 고지의 수정, 은폐 또는 삭제, 또는 (viii) 본 계약에 명시적으로 허가되지 않은 방식으로 소프트웨어 또는 설명서를 달리 사용 또는 복사하는 행위입니다.

### 2.3 배송.

GitHub는 고객이 암호로 보호된 안전한 웹사이트에서 다운로드할 수 있도록 라이선스 키를 제공할 것입니다. 본 섹션 2에 따른 모든 배송은 전자적 방식으로 이루어집니다. 의심의 여지를 피하기 위해 고객은 소프트웨어를 설치할 책임이 있으며, GitHub는 라이선스 키를 배송한 후 소프트웨어와 관련하여 추가적인 배송 의무가 없음을 인정합니다. 업데이트를 사용할 수 있게 되면 GitHub는 동일한 웹 사이트에서 다운로드하도록 제공할 것입니다. 고객은 연간 일(1) 회를 넘는 상업적으로 합리적인 간격으로 소프트웨어를 업데이트해야 합니다. 고객은 고객의 사용자 이름 및 암호에 대해 기밀을 유지할 책임이 있습니다.

### 2.4 인증.

**"공용 리포지토리"** 란 모든 사용자가 볼 수 있는 콘텐츠를 보관하는 리포지토리를 의미합니다.

**“릴리스”** 란 GitHub가 고객에게 설명서에 대한 해당 변경 내용과 함께 일반 공급하는 소프트웨어 릴리스를 의미하며, 일반적으로 첫 번째 소수점의 오른쪽 자리(예: x.x.x에서 x.y.x) 또는 첫 번째 소수점의 왼쪽 자리의 변경(예: x.x.x에서 y.x.x)으로 표시되는 개선 사항, 새로운 특징 또는 새로운 기능이 포함됩니다.

**“대리인”** 이란 당사자의 직원, 대리, 독립적 계약자, 컨설턴트, 법률 고문 및 재정 고문을 의미합니다.

**"스크래핑"** 이란 봇 또는 웹 크롤러와 같은 자동화된 프로세스를 통해 서비스에서 데이터를 추출하는 것을 의미하며, GitHub의 API를 통해 정보를 수집하는 것은 여기에 포함되지 않습니다.

**“서비스”** 란 호스팅된 GitHub Enterprise 클라우드 서비스를 의미합니다. 서비스에는 조직 계정, SAML Single Sign-On, 액세스 프로비저닝 및 해당 설명서가 포함됩니다. 이 기능 및 서비스 목록은 완전하지 않으며 수시로 업데이트될 수 있습니다.

**“서비스 크레딧”** 이란 아래 명시된 대로 계산되는 달러 크레딧을 의미하며, GitHub는 적격 계정에 크레딧을 적립해줄 수 있습니다.

**“소프트웨어”** 란 GitHub Enterprise Server 온-프레미스 소프트웨어를 의미합니다. 소프트웨어에는 GitHub Connect 기능, 모든 해당 설명서, GitHub가 고객에게 제공하거나 고객이 본 계약에 따라 고객의 구독에 포함된 경우 액세스할 수 있는 일체의 소프트웨어에 대한 모든 업데이트, 추가 기능 소프트웨어가 포함됩니다.

**“SOW”** 란 GitHub가 수행할 전문 서비스, 관련 요금 및 각 당사자의 관련 의무를 각각 자세히 설명하는 문서로서 상호 간에 작성한 작업 명세서를 의미합니다.

**“구독 라이선스”** 란 고객을 대신하여 제품을 설치, 운영, 액세스 및 사용하기 위해 각 사용자에게 할당된 라이선스를 의미합니다. 고객은 GitHub Enterprise Server 인스턴스 및 GitHub Enterprise 클라우드 조직에서 사용자당 하나의 구독 라이선스만 할당할 수 있습니다. 각 사용자는 고객이 허용하는 만큼의 고객 Enterprise Server 인스턴스 또는 Enterprise Cloud Organizations에 액세스할 수 있습니다. 그러나 명확성을 위해 고객이 한 사용자에게 구독 라이선스를 일단 할당하면, 한 명의 사용자가 Enterprise Server에서 구독 라이선스를 사용하는 동시에 다른 사용자가 GitHub Enterprise Server의 다른 인스턴스 또는 GitHub Enterprise Cloud의 조직에서 동일한 구독 라이선스를 사용할 수 있도록 고객이 구독 라이선스를 이분할 권한은 부여되지 않습니다.

**“구독 기간”** 이란 해당 주문 발효일 또는 주문서 양식에 달리 명시된 날로부터 일(1) 년을 의미합니다.

**“지원”** 이란 GitHub가 제공할 수 있는 소프트웨어 또는 서비스에 대한 기술 지원을 의미합니다.

**“업데이트”** 란 GitHub가 고객에게 설명서에 대한 해당 변경 내용과 함께 일반 공급하는 소프트웨어 릴리스를 의미하며, 일반적으로 첫 번째 소수점의 오른쪽 자리(예: x.x.x에서 x.y.x) 또는 첫 번째 소수점의 왼쪽 자리의 변경(예: x.x.x에서 y.x.x)으로 표시되는 오류 수정, 버그 수정이 포함됩니다.

**“작동 시간”** 이란 주어진 분기에 50%를 넘는 활성 사용자에게 영향을 미치는 작동 중단에 의해 GitHub의 필수 서비스가 중단되지 않은 시간의 비율을 의미합니다.

**“사용자”** 란 (i) 소프트웨어의 관점에서 소프트웨어의 실행을 개시하거나 소프트웨어의 기능 실행에서 소프트웨어와 상호작용하거나 지시를 내리는 단일 개인 또는 머신 계정, (ii) 서비스의 관점에서 다음의 각 경우에 고객을 대신하여 (a) 서비스에 액세스하거나 이를 사용하거나, (b) 고객 계정의 일부에 액세스하거나 이를 사용하거나, (c) 기능의 실행에서 고객의 계정 사용을 지시하는 개인 또는 머신 계정을 의미합니다. 사용자 수는 고객이 구입한 구독 라이선스의 수를 초과할 수 없습니다.

**"사용자 생성 콘텐츠"** 란 제3자 또는 외부 사용자가 만들거나 소유한 콘텐츠를 의미합니다.
