GitHub Enterprise 구독 계약을 참조하는 계약을 실행한 고객에게 적용됩니다. 모든 다른 기업 고객은 https://www.github.com/customer-terms 현행 약관으로 안내됩니다.

본 계약은 아래에 자세히 정의된 바와 같이 다음과 같은 GitHub 제품(총칭하여 **"제품"**)에 적용됩니다.

- 소프트웨어(Advanced Security, Insights 및 Learning Lab for Enterprise Server와 같은 추가 기능 소프트웨어를 포함할 수 있음)

- 관련된 모든 지원

- 관련된 모든 전문 서비스

고객이 GitHub 파트너에게 제품을 구입한 경우 본 계약의 다음 조항보다 고객과 GitHub 파트너 간의 계약조건이 우선합니다. 즉, 섹션 8, 결제, 섹션 9, 제공, 섹션 10, 인증 및 섹션 17, 기간 및 해지 조항입니다.

**1. 용어 정의.** 여기에 나열되지 않은 특정 용어는 본 계약 내에서 정의됩니다.

“**추가 기능 소프트웨어**”란 Advanced Security, Insights, Learning Lab for Enterprise Server 및 GitHub가 수시로 제공할 수 있는 기타 추가 소프트웨어 추가 기능 제품을 의미합니다.

“**Advanced Security**”란 고객이 사용자 지정 가능한 자동화된 시맨틱 코드 분석을 통해 보안 취약성을 식별하도록 지원하는 소프트웨어 기능입니다.

“**계약 발효일**”은 고객이 (i) 본 계약의 계약조건에 대해 “동의함” 단추를 클릭한 날짜 또는 (ii) 제품을 맨 처음 주문한 날짜 중 더 빠른 날짜입니다.

“**계열사**”란 당사자에게 직간접적으로 경영권을 행사하거나 당사자의 경영권 하에 있거나 당사자와 공동으로 경영권을 행사하는 모든 법인을 의미하며, 여기서 “경영권”이란 50% 이상의 소유권 또는 법인의 경영진에게 지시할 권리를 보유함을 의미합니다.

“**베타 미리 보기**”는 알파, 베타, 미리 보기, 사전 체험판, 평가판 또는 유사한 의미의 단어나 문구로 명시된 소프트웨어, 서비스 또는 기능을 의미합니다.

“**고객**”이란 “동의함” 단추 또는 이와 유사한 단추를 클릭하거나 해당 제품에 액세스하여 GitHub와 본 계약을 체결한 회사 또는 조직을 의미합니다.

“**고객 수정**”이란 버그 수정, 사용자 지정 또는 소프트웨어에 포함되거나 소프트웨어에 링크된 오픈 소스 라이선스 하에서 라이선스를 받은 라이브러리에 기능 추가를 하기 위한 목적만을 위해 고객이 수행한 소프트웨어 수정을 의미합니다.

“**설명서**”란 GitHub가 고객에게 제공하거나 전달하는 소프트웨어와 관련된 모든 설명서, 문서 및 기타 지원 자료를 의미합니다. 설명서는 소프트웨어의 일부로 간주됩니다.

“**피드백**”이란 GitHub 제품 또는 서비스에 관한 아이디어, 노하우, 알고리즘, 코드 기여, 제안, 개선 요청, 권장 사항 또는 그 밖의 모든 피드백을 의미합니다.

“**요금**”이란 해당 약정 기간 동안 제품 또는 전문 서비스를 사용하기 위해 고객이 GitHub에 지불해야 하는 요금을 의미하며, 이러한 요금은 주문서 양식 또는 SOW에 반영됩니다.

“**GitHub Insights**” 또는 “**Insights**”란 고객에게 소프트웨어 사용과 관련된 메트릭, 분석, 권고 사항을 제공하는 소프트웨어 기능을 의미합니다.  GitHub Insights에 조직 인사이트 및 리포지토리 인사이트를 비롯한 GitHub 레거시 기능은 포함되지 않습니다.

“**GitHub 파트너**”린 GitHub의 채널 파트너 계약의 계약조건에 따라 GitHub 제품을 재판매할 권한이 있는 회사를 의미합니다.

“**Learning Lab for Enterprise Server**”란 사용자가 관련 설명서를 포함하여 GitHub 기능에 대해 알아볼 수 있도록 지원하는 소프트웨어 기능을 의미합니다.

“**라이선스 발효일**”이란 본 계약에 명시된 각 주문 양식의 발효일을 의미합니다.

**“라이선스 키”** 란 소프트웨어를 설치, 작동 및 사용하도록 해주는 소프트웨어의 액세스 제어 메커니즘에 사용되는 데이터 파일을 의미하며, 암호로 보호된 보안 웹 사이트를 통해 제공됩니다.

“**주문서 양식**”이란 당사자가 제품을 주문하는 데 사용하는 서면 양식 또는 전자 문서(견적서 포함)를 의미합니다.

“**당사자**”란 GitHub 및 고객에 대한 언급을 통칭하는 의미입니다. 각각은 개별적으로 “**당사자**”라고 언급됩니다.

“**전문 서비스**”란 상호 간에 작성한 SOW에 따라 GitHub가 고객에게 제공하는 교육, 컨설팅 또는 구현 서비스를 의미합니다. 전문 서비스에는 지원이 포함되지 않습니다.

“**릴리스**”란 GitHub가 고객에게 설명서에 대한 해당 변경 내용과 함께 일반 공급하는 소프트웨어 릴리스를 의미하며, 일반적으로 첫 번째 소수점의 오른쪽 자리(예: x.x.x에서 x.y.x) 또는 첫 번째 소수점의 왼쪽 자리의 변경(예: x.x.x에서 y.x.x)으로 표시되는 개선 사항, 새로운 특징 또는 새로운 기능이 포함됩니다.

“**소프트웨어**”란 GitHub의 독점적인 Enterprise Server 소프트웨어를 의미합니다. 소프트웨어에는 모든 해당 설명서, GitHub가 고객에게 제공하거나 고객이 본 계약에 따라 고객의 구독에 포함된 경우 액세스할 수 있는 일체의 소프트웨어에 대한 모든 업데이트, 추가 기능 소프트웨어가 포함됩니다.

“**SOW**”란 GitHub가 수행할 전문 서비스, 관련 요금 및 각 당사자의 관련 의무를 각각 자세히 설명하는 문서로서 상호 간에 작성한 작업 명세서를 의미합니다.

“**구독 라이선스**”란 고객을 대신하여 소프트웨어를 설치, 운영, 액세스 및 사용하기 위해 각 사용자에게 할당된 라이선스를 의미합니다. 고객은 GitHub Enterprise Server 인스턴스에서 사용자당 하나의 구독 라이선스만 할당할 수 있습니다. 각 사용자는 고객이 허용하는 만큼의 고객 Enterprise Server 인스턴스에 액세스할 수 있습니다. 그러나 명확성을 위해 고객이 한 사용자에게 구독 라이선스를 일단 할당하면, 한 명의 사용자가 Enterprise Server에서 구독 라이선스를 사용하는 동시에 다른 사용자가 GitHub Enterprise Server의 다른 인스턴스에서 동일한 구독 라이선스를 사용할 수 있도록 고객이 구독 라이선스를 이분할 권한은 부여되지 않습니다.

“**구독 기간**”은 주문서 양식에 대체 기간을 지정하지 않는 한, 라이선스 발효일로부터 일(1) 년의 기간을 의미합니다. GitHub는 구독 기간이 만료되기 최소 60일 전에 갱신 알림을 고객에게 제공하고 갱신을 위한 새 주문서 양식을 생성합니다. GitHub는 주문서 양식에 따라 소프트웨어를 계속 사용하기 위해 고객이 다운로드할 수 있는 새로운 라이선스 키를 제공합니다.

“**지원**”이란 GitHub가 제공할 수 있는 소프트웨어에 대한 기술 지원을 의미합니다.

“**업데이트**”란 GitHub가 고객에게 설명서에 대한 해당 변경 내용과 함께 일반 공급하는 소프트웨어 릴리스를 의미하며, 일반적으로 첫 번째 소수점의 오른쪽 자리(예: x.x.x에서 x.y.x) 또는 첫 번째 소수점의 왼쪽 자리의 변경(예: x.x.x에서 y.x.x)으로 표시되는 오류 수정, 버그 수정이 포함됩니다.

“**사용자**”란 소프트웨어 실행을 시작하거나 소프트웨어의 기능 실행에 있어 소프트웨어와 상호 작용하거나 지시를 내리는 단일 사용자 또는 컴퓨터 계정을 의미합니다.

**2. 소프트웨어 라이선스 허여.** GitHub는 고객에게 고객의 내부 비즈니스 목적을 위해 해당 구독 기간 동안 설명서에 따라 고객의 주문서 양식에 명시된 구독 라이선스 개수에 대해서만 소프트웨어를 설치하고 사용할 수 있는 비배타적이며, 이전 불가하며, 전 세계적이며, 로열티가 없는, 기간 제한적인 라이선스를 허여합니다. 소프트웨어에는 제3자가 GitHub에 라이선스를 허여한 구성 요소가 포함되는데, 여기에는 GitHub가 해당 구성 요소에 대한 소스 코드를 제공하도록 요구하는 라이선스를 가진 소프트웨어가 포함됩니다. 해당 구성 요소에 대한 소스 코드는 요청 시 제공됩니다. 상기 내용을 제한하지 않고 본 라이선스는 고객이 Microsoft SQL Server Standard Edition Linux 파일용 컨테이너 이미지(“**SQL Server Images**”)를 다운로드하고 실행하도록 허가하며, 이는 문서화된 대로 소프트웨어와 함께만 사용할 수 있습니다. 고객이 SQL Server Images를 사용할 권한은 고객이 소프트웨어를 사용할 권한이 더 이상 없을 때 종료되며, 고객은 SQL Server Images의 사용 권한이 종료되면 이를 삭제해야 합니다. Microsoft Corporation은 언제든지 SQL Server Images를 사용 중지할 수 있습니다.

**3. 라이선스 제한.** 법률에 의해 또는 해당 제3자 라이선스에 의해 명시적으로 허가된 경우를 제외하고, 고객 및 그 계열사가 다음 행위를 하거나, 제3자가 하도록 허용해서는 안 됩니다. 즉, (i) 소프트웨어의 라이선스 재허여, 판매, 대여, 리스, 이전, 양도 또는 재배포, (ii) 제3자의 이익을 위한 소프트웨어 호스팅, (iii) 섹션 2에 명시적으로 허용한 경우를 제외하고 제3자가 소프트웨어에 액세스하도록 공개 또는 허가, (iv) 라이선스 키의 해킹이나 수정 또는 라이선스 등록 과정의 회피 또는 변경, (v) 고객 수정을 제외한 소프트웨어 파생 저작물 수정 또는 생성, 또는 소프트웨어의 다른 소프트웨어와의 병합, (vi) 소프트웨어의 분해, 디컴파일, 코드 이분 우회 또는 소프트웨어의 전체 또는 부분의 소스 코드 도출 시도, (vii) 소프트웨어 또는 설명서에 포함된 모든 독점 권리 고지의 수정, 은폐 또는 삭제, 또는 (viii) 본 계약에 명시적으로 허가되지 않은 방식으로 소프트웨어 또는 설명서를 달리 사용 또는 복사하는 행위입니다.

**4. 지적 재산권**. 양 당사자 가운데 GitHub는 제품에 대한 모든 지적 재산권을 포함하여 모든 권리, 소유권 및 이권을 소유합니다. GitHub는 본 계약에 따라 고객에게 명시적으로 부여되지 않은 권리로서 제품에 귀속된 모든 권리를 보유합니다.

**5. 피드백.** 고객은 제품에 관한 피드백을 GitHub에 제공할 수 있습니다. 피드백은 자발적이며 고객 비밀 정보에 속하지 않습니다(심지어 그와 같이 지정된 경우에도). GitHub는 (i) GitHub의 기존 및 향후 제품군의 운영, 기능 및 사용을 개선하고 그러한 제품군을 상용화하기 위해 그리고 (ii) 제품의 품질에 관하여 집계된 통계를 공개하기 위해 그러한 피드백을 충분히 행사하고 활용할 수 있습니다. 단, 그러한 공개 자료의 데이터는 고객과 그 직원 또는 고객의 독점 소프트웨어 코드를 구체적으로 식별하는 데 사용되지 않습니다.

**6. 구독 라이선스.** 구독 라이선스는 사용자별로 허여되며 여러 사용자가 동일한 구독 라이선스를 사용할 수는 없습니다. 라이선스 재할당의 원인이 (i) 하드웨어의 영구적인 오류나 손상, (ii) 사용자의 고용 또는 계약 해지, 또는 (iii) 사용자의 부재를 다루기 위한 구독 라이선스의 임시 재할당에서 비롯된 것이 아닌 한, 고객은 동일한 구독 라이선스를 마지막으로 재할당한 후 90일이 경과한 후에만 새 사용자에게 구독 라이선스를 재할당할 수 있습니다. 고객이 한 사용자의 구독 라이선스를 다른 사용자에게 재할당하는 경우, 고객은 구독 라이선스에 대한 이전 사용자의 액세스를 차단해야 합니다.

**7. 계열사.** 고객이 소프트웨어의 액세스 및 사용에 대해 여전히 전적인 책임을 지는 한, 고객의 계열사는 본 계약에 따라 소프트웨어를 사용할 권한이 있습니다.

**8. 결제.**

**9. 배송.** GitHub는 고객이 암호로 보호된 안전한 웹사이트에서 다운로드할 수 있도록 라이선스 키를 제공할 것입니다. 본 섹션 9에 따른 모든 배송은 전자적 방식으로 이루어집니다. 의심의 여지를 피하기 위해 고객은 소프트웨어를 설치할 책임이 있으며, GitHub는 라이선스 키를 배송한 후 소프트웨어와 관련하여 추가적인 배송 의무가 없음을 인정합니다. 업데이트를 사용할 수 있게 되면 GitHub는 동일한 웹 사이트에서 다운로드하도록 제공할 것입니다. 고객은 연간 일(1) 회를 넘는 상업적으로 합리적인 간격으로 소프트웨어를 업데이트해야 합니다. 고객은 고객의 사용자 이름 및 암호에 대해 기밀을 유지할 책임이 있습니다.

**14. 청구에 대한 방어, 면책**

당사자들은 이 섹션 14에 명시된 요건 및 범위 내에서 제3자 청구로부터 상대방을 방어하며 그로 인한 불리한 최종 판결 또는 승인된 합의의 금액을 지불하지만, 방어 당사자가 해당 청구에 대해 즉시 서면 통지를 받고 방어 및 그에 대한 합의를 통제할 수 있는 권리가 있는 경우에만 그러합니다. 방어를 받는 당사자는 방어하는 당사자에게 요청된 모든 지원, 정보 및 권한을 제공해야 합니다. 방어 당사자는 지원을 제공하는 과정에서 발생하는 합당한 현금 지출 비용을 상대 당사자에게 배상하며, 불합리하게 배상을 보류 또는 지연하지 않도록 상대 당사자의 사전 서면 동의 없이 제3자 청구와 관련하여 합의를 하거나 수락하지 않습니다. 이 섹션 14는 그러한 청구에 대한 양 당사자의 유일한 구제 수단과 전반적인 책임을 설명합니다.

**18.4** 정부 사용자. 제품 및 설명서는 전적으로 민간 자금으로 개발되었고 연방조달규정(Federal Acquisition Regulations) 12.212 및 27.405-3, 연방국방조달규정(Defense Federal Acquisition Regulation Supplement) 227.7202-3에서 규정한 “상용 컴퓨터 소프트웨어” 및 “상용 컴퓨터 소프트웨어 설명서”로 간주됩니다. 제품은 제한적 컴퓨터 소프트웨어 및 제한적 권한 데이터로 미국 정부 최종 사용자에 대해 라이선스가 허여됩니다. 이 계약 하에서 개발되는 기술 데이터 또는 컴퓨터 소프트웨어는 없습니다. 미국 정부 또는 그 계약자에 의한 제품 또는 설명서의 모든 사용, 공개, 수정, 배포 또는 복제는 본 계약에 명시된 제한 사항의 적용을 받습니다. 다른 모든 사용은 금지됩니다.

**18.5** 수출. 제품은 미국 정부가 관리하는 수출 및 제재와 특정 외국 정부의 수입 제한의 적용을 받으며, 고객은 제품을 사용하는 데 있어 모든 해당 수출입 법률과 규정을 준수해야 합니다. 고객은 제품의 어떤 부분 또는 일체의 직접적인 제품을 미국에서 제거하거나 다음으로 수출하거나, 또는 제3자가 그렇게 하도록 허용하거나, 수출 또는 재수출을 허용해서는 안 됩니다. 즉, (i) 수출 통제 대상 또는 테러리스트 지원 국가(또는 그 국적자 또는 거주자) 또는 영토에, (ii) 미국 상무부 부적격자 목록 또는 표에 있는 모든 사람, 특별 지정 제재 대상국 목록 또는 국가 “정부” 기관에 있는 사람 또는 기타 해당하는 금지 또는 제재 대상 개인 목록의 적용을 받는 모든 사람을 포함해 미국 해외자산관리청에서 관리하는 규칙에 따라 차단된 모든 사람에게, (iii) 해당 수출 또는 재수출이 제한 또는 금지된 국가 또는 미국 정부 또는 기관에서 수출 허가 또는 기타 정부 승인을 처음 획득하지 않은 경우 수출 또는 재수출 시 해당 허가나 승인을 요구하는 모든 국가 또는 영토에, 또는 (iv) 모든 미국 또는 해외 기관 또는 당국의 수출, 제재 또는 수입 제한, 법률 또는 규정을 달리 위반하여 수출 또는 재수출해서는 안 됩니다. 고객은 고객이 해당 금지 국가 또는 영토에 위치하거나 통제 하에 있거나 국적자 또는 거주자가 아니며 해당 금지 당사자 목록에 있지 않음을 진술하고 보증합니다. GitHub Enterprise Server는 고객의 자체 데이터 센터 또는 가상 개인 클라우드 내에서 실행할 수 있는 자체 호스팅 가상 어플라이언스입니다. 따라서 GitHub Enterprise Server를 사용하여 ITAR 또는 기타 내보내기 제어 정보를 저장할 수 있습니다. 그러나 고객은 해당 규정 준수를 보장할 책임이 있습니다. 고객은 제품이 미국 정부의 사전 필수 허가 없이 핵, 화학 또는 생물학 무기 또는 미사일 기술의 설계 또는 개발에 사용되지 못하도록 제한됨을 인정하고 이에 동의합니다.

**18.6** 허가 없는 홍보 금지. GitHub는 현재 고객 및 잠재 고객을 고객으로 식별할 수 있습니다. 그러나 GitHub는 고객의 이름 또는 로고를 고객의 허가 없이 광고 또는 마케팅 자료에 사용해서는 안 됩니다.

**18.7** 양도. 어떤 당사자도 상대 당사자의 사전 서면 동의 없이 본 계약의 전체 또는 일부를 양도하거나 달리 이전할 수 없고, 그러한 동의는 부당하게 보류되지 않아야 하며, 부당하게 동의를 보류하려는 시도는 무효로 간주됩니다. 단, 양도 당사자가 보유한 사업이나 자산의 전체 또는 실질적 전체의 합병, 인수, 기업 구조 개편 또는 매각과 관련하여 상대 당사자의 동의 없이 GitHub가 상대 당사자에게 통지하여 본 계약을 전부 양도할 수 있는 경우는 예외입니다.

**18.8** 통지. 본 계약에 달리 명시되지 않는 한, 본 계약에 따른 통지, 요청, 요구 또는 기타 통신은 서면으로 작성되어야 하며(메일은 허용 가능), 본 계약을 참조해야 하며, 다음과 같은 경우에 적절하게 제공된 것으로 간주됩니다. 즉, (i) 개인적으로 전달되는 경우, 수신 시 (ii) 메일의 경우 받는 사람이 수신을 확인한 후 영업일 기준 일(1) 일, (iii) 등기우편 또는 확인우편으로 보낸 경우 수령 서면 확인 및 메일과 함께 영업일 기준 오(5) 일, 또는 (iv) 국제적으로 인정받는 특급 택배 및 메일에 맡긴 경우 서면 수령 확인과 함께 영업일 기준 삼(3) 일입니다. 한 당사자가 해당 주소가 변경되었다는 사실을 상대방에게 알리지 않는 한, 본 계약에 명시된 주소로 통지를 전송할 수 있습니다.

**18.9** 불가항력. 불가항력, 천재지변, 파업, 봉쇄, 폭동, 전쟁행위, 전염병, 전력 공급 중단, 통신 장애, 네트워크 장애 등 통제할 수 없는 특별한 사유로 인하여 본 계약에 따른 의무를 이행할 수 없는 한, GitHub는 책임을 면합니다.

**18.10** 독립적 계약자. 당사자들은 본 계약의 주제와 관련하여 각각 독립적 계약자에 속합니다. 본 계약에 포함된 어떠한 조항도 당사자 간에 법인 단체, 파트너십(조합), 합작 투자, 고용, 대리인, 수탁자 또는 그 외 유사한 관계를 형성하는 것으로 간주되거나 해석되지 않으며 어떤 당사자도 상대방에게 계약에 따른 구속력을 행사할 수 없습니다.

**18.11** 권리 포기. 본 계약에 따른 한 당사자의 의무는 상대방의 승인된 대리인이 서명한 서면으로만 포기되어야 하며, 해당 포기는 명시된 특정 의무에 대해서만 유효합니다. 본 계약의 당사자가 본 계약에 따른 권리를 행사하지 않거나 지연하는 경우 이는 그러한 권리의 포기로 간주되지 않으며, 본 계약에 따른 권리를 단독으로 또는 부분적으로 행사하는 경우에도 그러한 권리의 또 다른 행사나 추가적인 행사 또는 법률 내지 형평법상 본 계약에 따른 권리의 행사를 제한하지 않습니다.

**18.12** 완전 합의. 각 주문서 양식 및 SOW를 포함한 본 계약은 주제 사안에 대한 당사자의 완전 합의와 양해를 구성하며 해당 주제 사안에 관해 구두 또는 서면에 관계없이 당사자들 간의 모든 과거 또는 동시적인 양해 및 합의에 우선합니다. 고객이 GitHub에 제출한 본 계약, 모든 SOW 또는 주문서 양식의 조건과 다르거나, 충돌하거나 이에 추가된 조건을 포함한 구매 주문서, 서면 약관 또는 기타 문서의 조건은 무효이며 효과가 없습니다.

**18.13** 개정, 우선순위. GitHub는 언제든지 본 계약의 계약조건을 수정할 수 있는 권리가 있으며 그러한 수정 사항이 있는 경우 본 계약을 업데이트합니다. 본 계약의 변경 내용은 고객의 당시 구독 기간의 1년이 되는 날짜에 발효됩니다. 본 계약과 주문서 양식이 충돌하는 경우 주문서 양식은 해당 주문에 대해서만 적용됩니다. 본 계약 및 주문서 양식을 포함한 모든 부록이 적용되어야 하며, 본 계약의 수정, 변경 또는 개정은 서면으로 작성되고 각 당사자의 승인된 대리인이 정당하게 서명하지 않는 한, 당사자들에게 구속력이 없거나 본 계약의 조건에 우선하지 못합니다.

**18.14** 분리가능성. 본 계약의 어떤 조항이 관할 법원에서 불법, 무효 또는 이행 불가능한 것으로 판정되는 경우, 법원은 해당 조항의 효력을 최대한 발생할 수 있도록 본 계약을 수정하거나 개정합니다. 이러한 방식으로 수정하거나 개정할 수 없는 모든 조항은 삭제된 것으로 간주되며 본 계약의 나머지 조항은 계속해서 완전한 효력을 유지합니다.
