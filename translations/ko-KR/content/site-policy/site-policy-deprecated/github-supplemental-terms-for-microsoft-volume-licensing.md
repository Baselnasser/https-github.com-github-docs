이러한 조건은 2021년 1월 4일 이전에 제품 라이선스를 받은 고객에게 적용됩니다. 해당 날짜 이후에 GitHub 제품을 구매한 고객은 https://www.github.com/enterprise-legal 의 현재 약관으로 안내됩니다.

다음 GitHub 보충 약관(모든 해당 주문서 양식 포함)은 고객의 Microsoft 볼륨 라이선싱 계약(“**Microsoft 고객 계약**”) 보충하며, Microsoft 고객 계약과 함께 고객의 제품 사용(아래 정의에 따름)을 규율합니다. Microsoft 고객 계약은 이 참조를 통해 본 계약에 통합됩니다.  본 보충 약관에서 정의되지 않고 사용된 특정 용어는 Microsoft 고객 계약에 할당된 의미로 사용됩니다.

본 보충 약관은 아래에 자세히 정의된 바와 같이 다음과 같은 GitHub 제품(총칭하여 “**제품**”)에 적용됩니다.

-   GitHub Enterprise(추가 기능 소프트웨어가 포함될 수 있는 GitHub Enterprise Server 및 GitHub Enterprise Cloud로 구성됨)

-   관련된 모든 지원

-   관련된 모든 전문 서비스

본 GitHub 보충 약관에는 다음 섹션과 별첨이 포함되며, 각각은 참조에 의해 본 계약에 포함됩니다.

* 섹션 1: GitHub Enterprise Server 라이선스 약관

* 섹션 2: GitHub Enterprise Cloud 서비스 약관

* 섹션 3: 일반 조항

* 별첨 A: 정의.



##  섹션 1: GitHub Enterprise Server 라이선스 약관

본 섹션 1에는 고객의 소프트웨어 사용에 적용되는 용어가 자세히 설명됩니다.

### 1.1 라이선스 허여.

GitHub는 고객에게 고객의 내부 비즈니스 목적을 위해 해당 구독 기간 동안 설명서에 따라 고객의 주문서 양식에 명시된 구독 라이선스 개수에 대해서만 소프트웨어를 설치하고 사용할 수 있는 비배타적이며, 이전 불가하며, 전 세계적이며, 로열티가 없는, 기간 제한적인 라이선스를 허여합니다. 소프트웨어에는 제3자가 GitHub에 라이선스를 허여한 구성 요소가 포함되는데, 여기에는 GitHub가 해당 구성 요소에 대한 소스 코드를 제공하도록 요구하는 라이선스를 가진 소프트웨어가 포함됩니다. 해당 구성 요소에 대한 소스 코드는 요청 시 제공됩니다. 상기 내용을 제한하지 않고 본 라이선스는 고객이 Microsoft SQL Server Standard Edition Linux 파일용 컨테이너 이미지(“**SQL Server Images**”)를 다운로드하고 실행하도록 허가하며, 이는 문서화된 대로 소프트웨어와 함께만 사용할 수 있습니다. 고객이 SQL Server Images를 사용할 권한은 고객이 소프트웨어를 사용할 권한이 더 이상 없을 때 종료되며, 고객은 SQL Server Images의 사용 권한이 종료되면 이를 삭제해야 합니다. Microsoft Corporation은 언제든지 SQL Server Images를 사용 중지할 수 있습니다.

### 1.2 제한 사항.

법률에 의해 또는 해당 제3자 라이선스에 의해 명시적으로 허가된 경우를 제외하고, 고객 및 그 계열사가 다음 행위를 하거나, 제3자가 하도록 허용해서는 안 됩니다. 즉, (i) 소프트웨어의 라이선스 재허여, 판매, 대여, 리스, 이전, 양도 또는 재배포, (ii) 제3자의 이익을 위한 소프트웨어 호스팅, (iii) 본 계약 섹션 1에 명시적으로 허용한 경우를 제외하고 제3자가 소프트웨어에 액세스하도록 공개 또는 허가, (iv) 라이선스 키의 해킹이나 수정 또는 라이선스 등록 과정의 회피 또는 변경, (v) 고객 수정을 제외한 소프트웨어 파생 저작물 수정 또는 생성, 또는 소프트웨어의 다른 소프트웨어와의 병합, (vi) 소프트웨어의 분해, 디컴파일, 코드 이분 우회 또는 소프트웨어의 전체 또는 부분의 소스 코드 도출 시도, (vii) 소프트웨어 또는 설명서에 포함된 모든 독점 권리 고지의 수정, 은폐 또는 삭제, 또는 (viii) 본 GitHub 보충 약관에 명시적으로 허가되지 않은 방식으로 소프트웨어 또는 설명서를 달리 사용 또는 복사하는 행위입니다.

### 1.3 배송.

GitHub는 고객이 암호로 보호된 안전한 웹사이트에서 다운로드할 수 있도록 라이선스 키를 제공할 것입니다. 본 섹션 1에 따른 모든 배송은 전자적 방식으로 이루어집니다. 의심의 여지를 피하기 위해 고객은 소프트웨어를 설치할 책임이 있으며, GitHub는 라이선스 키를 배송한 후 소프트웨어와 관련하여 추가적인 배송 의무가 없음을 인정합니다. 업데이트를 사용할 수 있게 되면 GitHub는 동일한 웹 사이트에서 다운로드하도록 제공할 것입니다. 고객은 연간 일(1) 회를 넘는 상업적으로 합리적인 간격으로 소프트웨어를 업데이트해야 합니다. 고객은 고객의 사용자 이름 및 암호에 대해 기밀을 유지할 책임이 있습니다.

### 1.4 인증.

GitHub의 요청 시, 고객은 고객이 본 GitHub 보충 약관에 따라 소프트웨어를 사용하고 있음을 인증하는 소프트웨어 생성 보고서를 GitHub에 즉시 제공합니다. GitHub는 GitHub 보충 약관 조건을 처음 초과하여 사용한 날짜부터 적용되는 추가 사용에 대해 고객에게 청구서를 발행합니다.

### 1.5 지원

GitHub는 Microsoft 고객 계약에 자세히 명시된 대로 소프트웨어에 대한 기술 지원을 제공합니다. GitHub는 [지원 약관](/enterprise/admin/enterprise-support/about-github-premium-support-for-github-enterprise) 및 주문서 양식 또는 SOW에 지정된 지원 수준, 요금 및 구독 기간에 따라 소프트웨어 및 서비스에 대한 향상된 지원 제품(프리미엄, 프리미엄 플러스 및 엔지니어링 직접 지원 제공 포함)을 제공할 수 있습니다. Microsoft 고객 계약에 반대됨에도 불구하고 (i) GitHub는 고객이 오류를 알릴 경우 소프트웨어의 실질적이고 재현 가능한 모든 오류를 수정하기 위해 합리적인 노력을 기울이겠지만 (a) (GitHub 이외의) 다른 사람이 소프트웨어를 수정한 경우, (b) 고객이 소프트웨어 또는 해당 성능에 부정적인 영향을 주는 방식으로 운영 체제 또는 환경을 변경한 경우, (c) 고객이 Microsoft 고객 계약, 본 섹션 1 또는 설명서에서 승인하지 않은 방식으로 소프트웨어를 사용한 경우, 또는 (d) 고객의 사고 또는 과실 또는 소프트웨어 오용이 있는 경우에 지원을 제공할 책임을 지지 않으며, (ii) GitHub는 원래 릴리스 날짜로부터 1년 또는 마지막 릴리스 업데이트로부터 6개월 동안 중에서 더 긴 동안만 해당 릴리스를 지원합니다.

### 1.6 업데이트, 릴리스.

#### 1.6.1 일반 조건.

GitHub는 고객이 소프트웨어 및 라이선스 키를 다운로드한 동일한 보안 웹사이트에서 소프트웨어에 대한 업데이트 및 릴리스를 고객에게 제공합니다.

#### 1.6.2 지원되는 릴리스.

GitHub는 원래 릴리스 날짜로부터 일(1) 년 동안 또는 릴리스의 마지막 업데이트로부터 육(6) 개월 동안에 해당하는 소프트웨어 릴리스만 지원합니다. 고객이 소프트웨어의 이전 릴리스에 대한 지원이 필요한 경우, 고객은 상호 합의된 주문서 양식 또는 SOW 조건에 따라 해당 지원에 대해 비용을 지불해야 합니다.

### 1.7 추가 기능 소프트웨어.

추가 기능 소프트웨어는 사용자당 기준으로 라이선스가 부여됩니다. 의심의 여지를 피하기 위하여 주문서 양식에 달리 명시되지 않는 한, 고객이 추가 기능 소프트웨어에 대해 지정된 시간에 보유한 구독 라이선스 수는 고객이 본 계약에 따라 제품에 대해 보유한 구독 라이선스 수와 같아야 합니다. 예를 들어 고객이 Insights에 대한 구독을 구매하고자 하고 해당 제품의 사용자 100명에 대한 구독 라이선스를 이미 보유하고 있는 경우 Insights 사용자 100명에 대한 구독 라이선스를 구매해야 합니다.

### 1.8 GitHub Insights 및 Learning Lab for Enterprise Server 사용 시 데이터 보호 고려 사항.

GitHub Insights 또는 Learning Lab for Enterprise Server에 대한 고객의 계획된 사용에 개인 데이터 처리가 수반되는 경우, 고객의 계획된 사용에 대해 데이터 보호 영향 평가를 수행할지 또는 공식적인 법적 분석을 확보할지를 판단하는 것은 전적으로 고객의 책임입니다. GitHub Insights 또는 Learning Lab for Enterprise Server를 사용하여 고객의 직원 및/또는 사용자의 데이터를 처리할지의 여부는 전적으로 고객의 재량이며, 고객이 그렇게 할 경우 해당 법률을 준수하여 그러한 처리를 수행할 책임은 전적으로 고객에게 있습니다.

### 1.9 제한된 소프트웨어 보증.

GitHub는 수정되지 않은 소프트웨어의 초기 다운로드가 제공된 날짜로부터 90일 동안 해당 소프트웨어가 설명서와 실질적으로 일치할 것을 보증합니다. GitHub는 고객의 소프트웨어 사용이 중단되지 않을 것임 또는 소프트웨어 작동에 오류가 없을 것임을 보증하지 않습니다. 이 보증은 고객이 본 섹션 1 및 설명서에 명시적으로 허용하지 않은 일체의 방식으로 소프트웨어를 수정 또는 사용하는 경우 적용되지 않습니다. GitHub 유일한 의무 및 이 제한된 보증 위반에 대한 고객의 유일한 구제책은 Microsoft 고객 계약에 명시된 내용에 따릅니다.

## 섹션 2: GitHub Enterprise Cloud 서비스 약관

서비스에 대해 고객이 또는 GitHub가 고객을 대신하여 기업 계정 및/또는 조직을 생성할 때, 본 섹션 2에 고객의 서비스 사용에 적용되는 조건이 자세히 설명됩니다.

### 2.1 계정 약관.

#### 2.1.1 계정 제어.

  **(i) 사용자.** 고객은 사용자가 자신의 개별 계정과 그 안에 있는 콘텐츠에 대한 최종 관리 권한을 보유한다는 점을 인정합니다. [GitHub의 표준 서비스 약관](/github/site-policy/github-terms-of-service)은 이 섹션 2에 따른 사용자의 활동을 제외하면 사용자의 서비스 이용에 적용됩니다.

  **(ii) 조직.** 고객은 이 섹션 2에 따라 고객을 대신하여 생성된 모든 조직 그리고 그 조직 내 리포지토리에 게시된 사용자 생성 콘텐츠에 대해 최종적인 관리 권한을 보유합니다. 이 섹션 2는 고객 조직의 사용에 적용됩니다.

#### 2.1.2 계정 요구 사항.

계정을 생성하기 위해 고객은 다음을 준수해야 합니다.

**(i)** 고객은 만 13세 미만의 어린이가 사용할 계정을 만들 수 없습니다. GitHub는 사용자가 만 13세 미만의 어린이임을 인지하면 그 사용자의 계정을 즉시 종료합니다. 고객 또는 그 사용자가 미국 이외의 국가에 있는 경우 해당 국가의 최소 연령 기준은 만 13세보다 더 높을 수 있습니다. 그러한 경우 고객은 해당 국가의 법률을 준수할 책임이 있습니다.

**(ii)** 사용자의 로그인은 여러 사람이 공유할 수 없습니다.

**(iii)** 고객은 다음과 같은 경우 서비스를 사용해서는 안 됩니다. 즉, (a) 미국 또는 그 외 해당 관할지의 수출 통제 또는 제재 법규를 위반하는 경우, (b) 미국 재무부 외국자산통제국(OFAC)이 시행하는 제재의 대상에 속한 국가 또는 영토에 소재하거나 상시 거주하는 경우, 또는 (c) 고객이 [특별제재대상국민(SDN)](https://www.treasury.gov/resource-center/sanctions/SDN-List/Pages/default.aspx)이거나 SDN을 대신하여 업무를 수행하는 경우 또는 그에 준하는 차단이나 거부 당사자 금지 대상에 해당하는 자인 경우입니다. 자세한 내용은 [GitHub의 무역 통제 정책](/github/site-policy/github-and-trade-controls)을 참조하시기 바랍니다.

#### 2.1.3 계정 보안.

고객은 다음에 대한 책임이 있습니다. 즉, (i) 게시된 모든 콘텐츠 및 기업 계정에서 발생하는 활동, (ii) 계정 로그인 자격 증명의 보안 유지, (iii) 계정을 통한 서비스의 무단 사용 또는 액세스를 알게 된 경우 이를 [즉시 GitHub에 통지](https://support.github.com/contact?tags=docs-policy) 등입니다.  GitHub는 고객이 이 섹션 2.1.3을 준수하지 않는 이유로 발생한 손실이나 손해에 대해 책임을 지지 않습니다.

#### 2.1.4 추가 조건.

일부 서비스 기능에는 [GitHub 추가 제품 약관](/github/site-policy/github-additional-product-terms)에 명시된 추가 약관이 적용될 수 있습니다. 이러한 기능에 액세스하거나 이 기능을 사용함으로써 고객은 GitHub 추가 제품 약관에 동의합니다.

## 섹션 3: 일반 조항.

본 섹션 3은 고객의 구매 및 제품 사용에 적용되는 약관을 명시합니다.

### 3.1 기간, 해지, 해지의 효력.

#### 3.1.1 기간.
본 GitHub 보충 약관은 발효일에 시작되어 이 섹션 3.1에 따라 당사자가 해지할 때까지 그 효력을 계속 유지합니다.

#### 3.1.2 편의상의 해지, 계정 취소.
각 당사자는 해당 구독 기간이 끝나기 전에 최소 30일 기한의 사전 서면 통지를 통해 특별한 사유 없이 주문서 양식 또는 본 GitHub 보충 약관을 해지할 수 있습니다. 고객이 주문서 양식 또는 본 GitHub 보충 약관을 해지하기로 선택한 경우, 화면 상단의 전역 탐색 모음에서 Settings(설정)로 이동하여 GitHub 계정을 적절하게 취소할 책임은 고객에게 있습니다. GitHub는 이메일 또는 전화 요청에 대한 응답으로 계정을 취소할 수 없습니다.

#### 3.1.3. 중대한 위반에 따른 해지.
각 당사자는 계약 상대방이 본 GitHub 보충 약관에 따른 중대한 의무를 위반하고 통지를 받은 날로부터 30일 이내에 위반을 시정하지 않으면 통지 즉시 본 GitHub 보충 약관을 해지할 수 있습니다.  GitHub는 고객의 계정이 90일 이상 정지된 경우 본 GitHub 보충 약관을 해지할 수 있습니다.

#### 3.1.4 해지의 효력.
본 GitHub 보충 약관이 해지되면 고객은 추가 주문서 양식을 작성할 수 없습니다. 다만 본 GitHub 보충 약관은 활성 주문서 양식의 나머지 약정 기간 동안 유효합니다. 주문서 양식의 구독이 해지되거나 만료되는 경우 동 양식과 관련하여 (i) 구독 기간은 즉시 종료되고, (ii) 주문서 양식의 모든 구독 라이선스는 자동으로 해지되며 고객은 더 이상 제품을 사용할 권리가 없으며, (iii) 구독 해지 전에 지불해야 할 요금이 남아 있다면 고객은 해당 요금을 즉시 지불해야 하며, (iv) 고객은 고객이 소지하거나 제어하고 있는 모든 소프트웨어 카피를 파기하고 고객이 그렇게 했음을 GitHub에 서면으로 확인해야 하며, (v) 각 당사자는 서비스가 허용하는 범위 내에서 상대방에게 속한 모든 비밀 정보를 즉시 반환(또는 상대방이 요청할 경우 파기)합니다. 상기 내용에도 불구하고 고객은 고객의 데이터를 마이그레이션하는 소프트웨어에 액세스할 수 있고, 본 계약 또는 주문서 양식의 해지 또는 만료 이후 최대 구십(90) 일까지 고객의 리포지토리에 있는 데이터 마이그레이션을 요청할 수 있습니다. 하지만 고객은 해당 시간 동안 생산을 근거로 소프트웨어나 서비스를 사용해서는 안 됩니다. 성격상 합리적으로 효력이 유지되어야 하는 모든 조항은 본 계약 또는 주문서 양식의 해지 또는 만료 후에도 효력이 유지됩니다.

### 3.2 피드백.
고객은 제품에 관한 피드백을 GitHub에 제공할 수 있습니다. 피드백은 자발적이며 고객 비밀 정보에 속하지 않습니다(심지어 그와 같이 지정된 경우에도). GitHub는 (i) GitHub의 기존 및 향후 제품군의 운영, 기능 및 사용을 개선하고 그러한 제품군을 상용화하기 위해 그리고 (ii) 제품의 품질에 관하여 집계된 통계를 공개하기 위해 그러한 피드백을 충분히 행사하고 활용할 수 있습니다. 단, 그러한 공개 자료의 데이터는 고객과 그 직원 또는 고객의 독점 소프트웨어 코드를 구체적으로 식별하는 데 사용되지 않습니다.

### 3.3 법률 및 규정 준수.
고객은 제품 사용에 있어서 데이터 보호 및 고용 법률과 규정을 포함하지만 이에 국한되지 않는 모든 관련 법률 및 규정을 준수합니다.

### 3.4 우선 순위
반면에 보충 약관과 주문서 양식이 충돌하는 경우 주문서 양식은 해당 주문에 대해서만 적용됩니다. 보충 약관(주문서 양식 포함)과 Microsoft 고객 계약 간에 충돌이 발생하는 경우, 보충 약관은 해당 주제 사안에 대해서만 적용됩니다.

**“활성 사용자”** 란 작동 중단 시 서비스에 액세스를 시도하는 사용자를 의미합니다.

**“요금”** 이란 (i) 해당 구독 기간 동안 제품을 사용하거나 (ii) 전문 서비스를 받기 위해 고객이 GitHub에 지불해야 하는 요금을 의미하며, 이러한 요금은 주문서 양식 또는 SOW에 반영됩니다.

**“피드백”** 이란 GitHub 제품 또는 서비스에 관한 아이디어, 노하우, 알고리즘, 코드 기여, 제안, 개선 요청, 권장 사항 또는 그 밖의 모든 피드백을 의미합니다.

**"포크"** 란 한 리포지토리의 콘텐츠를 다른 리포지토리로 복사하는 것을 의미합니다.

**“GitHub”** 란 통칭하여 GitHub, Inc., 그 계열사 및 대리인을 의미합니다.

**"GitHub 콘텐츠"** 란 GitHub가 생성하고 소유하거나 GitHub가 권리를 보유하는 콘텐츠를 의미합니다.

**“GitHub Insights”** 또는 **“Insights”** 란 고객에게 소프트웨어 사용과 관련된 메트릭, 분석, 권고 사항을 제공하는 소프트웨어 기능을 의미합니다. GitHub Insights에 조직 인사이트 및 리포지토리 인사이트를 비롯한 GitHub 레거시 기능은 포함되지 않습니다.

**“Learning Lab for Enterprise Server”** 란 사용자가 관련 설명서를 포함하여 GitHub 기능에 대해 알아볼 수 있도록 지원하는 소프트웨어 기능을 의미합니다.

**“라이선스 키”** 란 고객이 소프트웨어를 설치, 작동 및 사용하도록 해주는 소프트웨어의 액세스 제어 메커니즘에 사용되는 데이터 파일입니다.

**"컴퓨터 계정"** 이란 컴퓨터 계정을 위한 해당 서비스 약관을 수락하고 유효한 이메일 주소를 제공하며 해당 작업에 대한 책임이 있는 개인이 등록한 계정을 의미합니다. 컴퓨터 계정은 자동화된 작업을 수행하는 데만 사용됩니다. 여러 사용자가 컴퓨터 계정의 작업을 지시할 수 있지만 컴퓨터의 작업에 대한 최종적인 책임은 해당 계정의 소유자에게 있습니다.

**“주문서 양식”** 이란 당사자가 제품을 주문하는 데 사용하는 서면 양식 또는 전자 문서(견적서 포함)를 의미합니다.

**"조직"** 이란 여러 사용자가 한 번에 여러 프로젝트에서 협업할 수 있는 경우 단일 기관 또는 한 명 이상의 사용자와 연결될 수 있는 공유 작업 공간을 의미합니다. 사용자는 둘 이상의 조직에 속한 구성원이 될 수 있습니다.

**“작동 중단”** 이란 50%를 넘는 활성 사용자에게 영향을 미치는 필수 서비스의 중단을 의미합니다.

**"개인 리포지토리"** 란 사용자가 콘텐츠에 대한 액세스를 제어할 수 있는 리포지토리를 의미합니다.

**“전문 서비스”** 란 상호 간에 작성한 SOW에 따라 GitHub가 제공하는 교육, 컨설팅 또는 구현 서비스를 의미합니다. 전문 서비스에는 지원이 포함되지 않습니다.

**"공용 리포지토리"** 란 모든 사용자가 볼 수 있는 콘텐츠를 보관하는 리포지토리를 의미합니다.

**“릴리스”** 란 GitHub가 고객에게 설명서에 대한 해당 변경 내용과 함께 일반 공급하는 소프트웨어 릴리스를 의미하며, 일반적으로 첫 번째 소수점의 오른쪽 자리(예: x.x.x에서 x.y.x) 또는 첫 번째 소수점의 왼쪽 자리의 변경(예: x.x.x에서 y.x.x)으로 표시되는 개선 사항, 새로운 특징 또는 새로운 기능이 포함됩니다.

**“대리인”** 이란 당사자의 직원, 대리, 독립적 계약자, 컨설턴트, 법률 고문 및 재정 고문을 의미합니다.

**"스크래핑"** 이란 봇 또는 웹 크롤러와 같은 자동화된 프로세스를 통해 서비스에서 데이터를 추출하는 것을 의미하며, GitHub의 API를 통해 정보를 수집하는 것은 여기에 포함되지 않습니다.

**“서비스”** 란 호스팅된 GitHub Enterprise 클라우드 서비스를 의미합니다. 서비스에는 조직 계정, SAML Single Sign-On, 액세스 프로비저닝 및 해당 설명서가 포함됩니다. 이 기능 및 서비스 목록은 완전하지 않으며 수시로 업데이트될 수 있습니다.

**“서비스 크레딧”** 이란 아래 명시된 대로 계산되는 달러 크레딧을 의미하며, GitHub는 적격 계정에 크레딧을 적립해줄 수 있습니다.

**“소프트웨어”** 란 GitHub Enterprise Server 온-프레미스 소프트웨어를 의미합니다. 소프트웨어에는 GitHub Connect 기능, 모든 해당 설명서, GitHub가 고객에게 제공하거나 고객이 본 GitHub 보충 약관에 따라 고객의 구독에 포함된 경우 액세스할 수 있는 일체의 소프트웨어에 대한 모든 업데이트, 추가 기능 소프트웨어가 포함됩니다.

**“SOW”** 란 GitHub가 수행할 전문 서비스, 관련 요금 및 각 당사자의 관련 의무를 각각 자세히 설명하는 문서로서 상호 간에 작성한 작업 명세서를 의미합니다.

**“구독 라이선스”** 란 고객을 대신하여 제품을 설치, 운영, 액세스 및 사용하기 위해 각 사용자에게 할당된 라이선스를 의미합니다. 고객은 GitHub Enterprise Server 인스턴스 및 GitHub Enterprise 클라우드 조직에서 사용자당 하나의 구독 라이선스만 할당할 수 있습니다. 각 사용자는 고객이 허용하는 만큼의 고객 Enterprise Server 인스턴스 또는 Enterprise Cloud Organizations에 액세스할 수 있습니다. 그러나 명확성을 위해 고객이 한 사용자에게 구독 라이선스를 일단 할당하면, 한 명의 사용자가 Enterprise Server에서 구독 라이선스를 사용하는 동시에 다른 사용자가 GitHub Enterprise Server의 다른 인스턴스 또는 GitHub Enterprise Cloud의 조직에서 동일한 구독 라이선스를 사용할 수 있도록 고객이 구독 라이선스를 이분할 권한은 부여되지 않습니다. 구독 라이선스는 사용자별로 허여되며 여러 사용자가 동일한 구독 라이선스를 사용할 수는 없습니다. 라이선스 재할당의 원인이 (i) 하드웨어의 영구적인 오류나 손상, (ii) 사용자의 고용 또는 계약 해지, 또는 (iii) 사용자의 부재를 다루기 위한 구독 라이선스의 임시 재할당에서 비롯된 것이 아닌 한, 고객은 동일한 구독 라이선스를 마지막으로 재할당한 후 90일이 경과한 후에만 새 사용자에게 구독 라이선스를 재할당할 수 있습니다. 고객이 한 사용자의 구독 라이선스를 다른 사용자에게 재할당하는 경우, 고객은 구독 라이선스 및 고객의 조직에 대한 이전 사용자의 액세스를 차단해야 합니다.

**“구독 기간”** 이란 해당 주문 발효일 또는 주문서 양식에 달리 명시된 날로부터 일(1) 년을 의미합니다.

**“지원”** 이란 GitHub가 제공할 수 있는 소프트웨어 또는 서비스에 대한 기술 지원을 의미합니다.

**“업데이트”** 란 GitHub가 고객에게 설명서에 대한 해당 변경 내용과 함께 일반 공급하는 소프트웨어 릴리스를 의미하며, 일반적으로 첫 번째 소수점의 오른쪽 자리(예: x.x.x에서 x.y.x) 또는 첫 번째 소수점의 왼쪽 자리의 변경(예: x.x.x에서 y.x.x)으로 표시되는 오류 수정, 버그 수정이 포함됩니다.

**“작동 시간”** 이란 주어진 분기에 50%를 넘는 활성 사용자에게 영향을 미치는 작동 중단에 의해 GitHub의 필수 서비스가 중단되지 않은 시간의 비율을 의미합니다.

**“사용자”** 란 (i) 소프트웨어의 관점에서 소프트웨어의 실행을 개시하거나 소프트웨어의 기능 실행에서 소프트웨어와 상호작용하거나 지시를 내리는 단일 개인 또는 머신 계정, (ii) 서비스의 관점에서 다음의 각 경우에 고객을 대신하여 (a) 서비스에 액세스하거나 이를 사용하거나, (b) 고객 계정의 일부에 액세스하거나 이를 사용하거나, (c) 기능의 실행에서 고객의 계정 사용을 지시하는 개인 또는 머신 계정을 의미합니다. 사용자 수는 고객이 구입한 구독 라이선스의 수를 초과할 수 없습니다.

**"사용자 생성 콘텐츠"** 란 제3자 또는 외부 사용자가 만들거나 소유한 콘텐츠를 의미합니다.
