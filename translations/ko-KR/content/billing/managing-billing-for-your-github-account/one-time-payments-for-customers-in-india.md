## Reserve Bank of India의 반복 지불 규제 정보

RBI(Reserve Bank of India)의 새로운 지불 규정이 최근 발효되었습니다. 이 규정은 반복적인 온라인 거래에 추가 요구 사항을 부여하며 인도의 일부 {% data variables.product.company_short %} 고객이 반복적으로 결제하지 못하도록 합니다. {% data variables.product.product_name %}의 반복 거래에 대해 인도에서 발행된 결제 방법을 사용하는 고객은 은행이나 카드 발급자가 결제를 거절할 수 있습니다. 자세한 내용은 [RBI 보도 자료](https://www.rbi.org.in/Scripts/BS_PressReleaseDisplay.aspx?prid=51353)를 참조하세요.

규정은 다음을 비롯한 모든 반복 거래에 적용됩니다.
- {% data variables.product.prodname_dotcom %} 플랜 구독(Pro, Team, Enterprise)
- {% data variables.product.prodname_marketplace %} 구매
- {% data variables.product.prodname_sponsors %} 거래
- Git 대형 파일 스토리지 구매
- {% data variables.product.prodname_actions %}, {% data variables.product.prodname_registry %} 및 {% data variables.product.prodname_github_codespaces %} 소비

중단을 최소화하기 위해 영향을 받는 고객에 대한 반복 결제가 2021년 10월 29일에 일시 중지되었습니다. 유료 기능 및 서비스는 RBI 규정의 영향을 받는 고객에게 계속 제공됩니다.

## {% data variables.product.company_short %}에 대한 일회성 결제 정보

새로운 요구 사항을 충족하기 위해 결제 게이트웨이 공급자와 협력하면서 영향을 받은 인도의 고객을 위한 임시 일회성 결제 옵션을 제공하고 있습니다. 2022년 2월 15일부터 새로운 RBI 규정의 영향을 받은 인도의 {% data variables.product.company_short %} 고객은 일반 청구 주기 보조에 따라 일회성 결제를 할 수 있습니다.

### 월별 청구 고객의 경우

월별 청구 플랜의 고객은 보통 청구 주기가 갱신되는 날에 일회성 결제를 할 수 있습니다. 예를 들어 보통 매월 7일에 청구된다면 매월 7일부터 계정에서 일회성 결제를 할 수 있습니다. 첫 번째 일회성 결제에는 2021년 10월부터 발생한 사용량도 포함됩니다.

현재 월별로 청구되고 연간 청구로 전환하려는 경우 일회성 결제 빈도를 줄일 수 있습니다. 자세한 내용은 “[청구 기간 변경](/en/billing/managing-your-github-billing-settings/changing-the-duration-of-your-billing-cycle)”을 참조하세요.

### 연간 청구 고객의 경우

요금이 연 단위로 청구되고 갱신 날짜가 2021년 10월 1일부터 2022년 2월 14일 사이인 경우 2월 15일부터 연간 구독에 대해 일회성 결제를 할 수 있습니다. 이 초기 결제에는 이전 청구 주기가 종료된 이후 기간 동안 구독의 비례 배분된 미결제 비용이 포함됩니다.

청구 주기가 2월 15일 이후에 갱신될 예정인 경우 반복 결제를 시도합니다. 결제 시도가 거부되면 계정의 청구 페이지를 통해 일회성 결제를 할 수 있습니다.

그동안, 영향을 받은 고객에 대한 반복 지불을 복구하기 위해 지불 파트너와 적극적으로 협력하고 있습니다. 자세한 내용이나 질문은 [GitHub 지원](https://support.github.com/contact)에 문의하세요.

### {% data variables.product.prodname_sponsors %}에 미치는 영향

기존 스폰서쉽은 이 기간 동안 그대로 유지되며 유지 보수는 예정대로 계속 지급됩니다. 기금 계정에서 발생한 스폰서쉽 금액에 대한 지불은 다른 누적 요금과 동시에 수집됩니다.
