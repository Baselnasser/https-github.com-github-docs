유료 개인 계정 또는 조직 구독에 대한 변경 내용은 다른 유료 {% data variables.product.prodname_dotcom %} 기능(예: {% data variables.large_files.product_name_long %} 또는 {% data variables.product.prodname_marketplace %}에서 구매한 유료 앱)에 대한 구독 또는 결제에 영향을 미치지 않습니다.

자세한 내용은 “[{% data variables.product.prodname_dotcom %}의 제품](/articles/github-s-products)” 및 “[{% data variables.product.prodname_dotcom %}에 대한 청구 정보](/articles/about-billing-on-github)”를 참조하세요.

## 개인 계정 또는 조직에 대한 유료 구독을 취소하는 예제

Kumiko는 매월 5일에 월간 구독료를 지불합니다. Kumiko가 10월 10일에 유료 구독에서 {% data variables.product.prodname_free_user %}로 다운그레이드하는 경우 유료 구독은 11월 4일에 현재 청구 주기가 끝날 때까지 계속 적용됩니다. 11월 5일에 계정이 {% data variables.product.prodname_free_user %}로 이동합니다.

## 개인 계정 또는 조직을 연간 구독에서 월간 구독으로 변경하는 예제

Ravi는 매년 10월 5일에 연간 구독료를 지불합니다. Ravi가 12월 10일에 연간 청구에서 월간 청구로 전환하는 경우, Ravi의 계정은 다음 해 10월 4일에 현재 청구 주기가 끝날 때까지 연간 구독이 유지됩니다. 다음 해 10월 5일에는 Ravi에게 한 달에 대한 서비스 요금이 청구됩니다. Ravi의 다음 청구 날짜는 11월 5일입니다.

## 조직에 유료 사용자를 추가하는 예제

Mada의 조직은 매월 15일에 사용자 25명에 대한 요금을 지불합니다. Mada가 6월 4일에 유료 사용자 10명을 추가하면 6월 4일부터 6월 14일까지 추가 사용자 10명에 대한 비례 배분 요금이 즉시 청구되며, 바로 사용할 수 있습니다. 6월 15일에 Mada의 조직은 사용자 35명에 대한 요금을 지불합니다.

## 조직에서 유료 사용자를 제거하는 예제

Stefan의 조직은 5월 20일에 매년 사용자 50명에 대한 요금을 지불합니다. Stefan이 9월 30일에 사용자 20명을 제거하고 총 30명의 유료 사용자로 다운그레이드할 경우, 조직은 5월 19일에 현재 청구 주기가 끝날 때까지 유료 사용자 50명에게 계속 액세스할 수 있습니다. 5월 20일에 다운그레이드가 적용되면 Stefan의 조직은 30명에 대한 요금을 지불하고 유료 사용자 30명에게 액세스할 수 있습니다.
