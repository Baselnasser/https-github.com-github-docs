## 퍼블릭 리포지토리의 지속성

{% data variables.product.product_name %}는 퍼블릭 리포지토리를 제거하지 않는 한 사용 가능한 상태로 유지하려고 합니다. 경우에 따라 퍼블릭 콘텐츠를 사용할 수 없게 만들 수 있습니다. 예를 들면 다음과 같은 경우입니다.

- 리포지토리의 콘텐츠에 대한 [DMCA 테이크다운 알림](/free-pro-team@latest/github/site-policy/dmca-takedown-policy)이 표시되는 경우.
- 리포지토리의 콘텐츠가 [커뮤니티 지침](/free-pro-team@latest/github/site-policy/github-community-guidelines) 또는 [서비스 약관](/free-pro-team@latest/github/site-policy/github-terms-of-service)을 위반하는 것으로 판단되는 경우.

학계와 연구원은 데이터 관리 계획에서 이 정보를 참조할 수 있습니다.

## {% data variables.product.prodname_archive %} 정보

{% data reusables.repositories.about-github-archive-program %}

{% data variables.product.prodname_archive %}에서는 타사 파트너가 퍼블릭 API를 사용하여 퍼블릭 리포지토리를 보관할 수 있습니다. 이러한 파트너는 다양한 형식의 데이터를 다양한 빈도로 보관하고 데이터를 공개적으로 사용할 수 있게 만듭니다. 또한 {% data variables.product.prodname_archive %}은 다양한 데이터 형식 및 위치에 여러 복사본을 저장하여 지속적으로 데이터를 보호합니다. 예를 들어 {% data variables.product.company_short %}는 1,000년 이상 지속되도록 의도된 매우 장기적인 아카이브인 {% data variables.product.prodname_arctic_vault %}에 리포지토리를 저장합니다. 자세한 내용은 [{% data variables.product.prodname_archive %}](https://archiveprogram.github.com/)을 참조하세요.

아카이브의 책임 있는 사용에는 사용자의 개인 정보 보호가 포함됩니다. 자세한 내용은 “[GitHub 대한 공개 정보](/free-pro-team@latest/github/site-policy/github-privacy-statement#public-information-on-github)”를 참조하세요.

리포지토리에 대해 {% data variables.product.prodname_archive %}을 옵트아웃할 수 있습니다. 자세한 내용은 “[퍼블릭 리포지토리에 대해 {% data variables.product.prodname_archive %} 옵트인 또는 옵트아웃](/get-started/privacy-on-github/opting-into-or-out-of-the-github-archive-program-for-your-public-repository)”을 참조하세요.

## 보관성을 높이기 위해 오픈 소스 라이선스 추가

라이브러리 및 연구원은 공개적으로 사용 가능한 콘텐츠의 아카이브를 만들기 위해 법적 보호가 필요할 수 있습니다. 타사에서 내 작업의 보관을 위해 {% data variables.product.product_name %}를 고려하도록 하려는 경우 내 프로젝트에 [오픈 소스 라이선스](/articles/open-source-licensing)를 추가할 수 있습니다. 오픈 소스 라이선스는 기여자에게 리포지토리에 자료를 복사하고 배포할 수 있는 명시적 권한을 부여합니다.
