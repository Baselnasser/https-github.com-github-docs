비밀을 푸시하도록 허용하면 “보안” 탭에 경고가 생성됩니다. {% data variables.product.prodname_dotcom %}는 경고를 닫고 비밀이 가양성이거나 테스트에서만 사용되도록 지정하는 경우 알림을 보내지 않습니다. 비밀이 실제이고 나중에 수정하도록 지정하면 {% data variables.product.prodname_dotcom %}는 보안 경고를 열어 두고 커밋 작성자와 리포지토리 관리자에게 알림을 보냅니다. 자세한 내용은 “[비밀 검사에서 경고 관리](/code-security/secret-scanning/managing-alerts-from-secret-scanning)”를 참조하세요.
