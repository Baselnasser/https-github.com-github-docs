# 제품 랜딩 예제

`product-landing` 레이아웃을 사용하는 페이지에는 선택적으로 `Examples` 섹션을 포함할 수 있습니다. 현재, 다음과 같은 세 가지 형식을 지원합니다.

1. 코드 예제 https://docs.github.com/en/codespaces#code-examples 를 참조하세요.

2. Community 예제 https://docs.github.com/en/discussions#community-examples 를 참조하세요.

3. 사용자 예제 https://docs.github.com/en/sponsors#community-examples 를 참조하세요.

## 작동 방식

각 제품에 대한 예제 데이터는 `data/product-landing-examples`, **제품** 에 대해 명명된 하위 디렉터리 및 **예제 유형** 에 대해 명명된 YML 파일에서 정의됩니다(예: `data/product-examples/sponsors/user-examples.yml` 또는`data/product-examples/codespaces/code-examples.yml`). 현재 제품당 한 가지 유형의 예제만 지원합니다.

### 버전 관리

현재 버전 관리는 코드 예제에서만 지원됩니다. **모든** 버전에서 예제 블록을 사용할 수 있어야 하는 경우 특별한 작업을 수행할 필요가 없습니다. 그러나 예제 블록을 일부 버전에서만 사용할 수 있어야 하는 경우 다음과 같은 `versions` prop를 추가할 수 있습니다.

```
- title: Dependabot version update PR
  description: Example pull request generated by the Dependabot version updates configuration in the Super linter repository.
  href: /github/super-linter/pull/1398
  languages:
  tags:
    - Dependabot
    - Version updates
    - Pull requests
  versions:
    fpt: '*'
  ```

여기서 `versions`에 대한 구문은 [frontmatter `versions` 속성](content/README.md)과 동일하며 SemVer 표기법을 지원할 수 있습니다.

## 렌더링

제품 예제 데이터가 `middleware/contextualizers/product-examples.js`의 `context` 개체에 추가됩니다.

그러면 `components/landing`이 데이터를 렌더링합니다.
