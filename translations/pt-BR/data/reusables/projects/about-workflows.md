Os {% data variables.product.prodname_projects_v2 %} incluem fluxos de trabalho internos que você pode usar para atualizar o **Status** dos itens com base em determinados eventos. Por exemplo, você pode definir automaticamente o status como **Tarefas pendentes** quando um item é adicionado ao projeto ou definir o status como **Concluído** quando um problema é fechado.

Quando o projeto é inicializado, dois fluxos de trabalho são habilitados por padrão: quando problemas ou solicitações de pull no seu projeto são fechados, o status deles é definido como **Concluído**, e quando solicitações de pull no seu projeto são mescladas, o status delas é definido como **Concluído**.
