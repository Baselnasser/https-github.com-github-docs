Quando você usa um IdP para IAM em {% data variables.product.product_name %}, o SSO SAML controla e protege o acesso a recursos corporativos, como repositórios, problemas e solicitações de pull. O SCIM cria automaticamente contas de usuário e gerencia o acesso ao {% data variables.location.product_location %} quando você faz alterações no IdP. Você também pode sincronizar equipes em {% data variables.product.product_name %} com grupos em seu IdP. Para obter mais informações, consulte os seguintes artigos.

- "[Sobre o SAML para IAM corporativo](/admin/identity-and-access-management/using-saml-for-enterprise-iam/about-saml-for-enterprise-iam)"
- "[Como configurar provisionamento de usuário com SCIM para sua empresa](/admin/identity-and-access-management/using-saml-for-enterprise-iam/configuring-user-provisioning-with-scim-for-your-enterprise)"
