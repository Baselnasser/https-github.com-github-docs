Os ambientes são usados para descrever um destino de implantação geral, como `production`, `staging` ou `development`. Quando um fluxo de trabalho de {% data variables.product.prodname_actions %} é implantado em um ambiente, o ambiente é exibido na página principal do repositório. Você pode usar ambientes para exigir aprovação para um trabalho para prosseguir, restringir quais branches podem acionar um fluxo de trabalho ou limitar o acesso a segredos. Para obter mais informações sobre como criar ambientes, confira "[Como usar ambientes para implantação](/actions/deployment/using-environments-for-deployment)".
