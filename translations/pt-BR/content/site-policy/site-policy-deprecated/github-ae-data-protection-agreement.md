_Estes termos se aplicam aos Clientes que licenciaram os Produtos antes de 4 de janeiro de 2021. Clientes que comprarem produtos GitHub após essa data serão direcionados para https://www.github.com/enterprise-legal para consultarem os termos atuais._

## INTRODUÇÃO

As partes concordam que o Contrato de Proteção de Dados do GitHub AE e o Anexo de segurança (coletivamente, o "**Contrato de Proteção de Dados**" ou "**DPA**") definem obrigações no que diz respeito ao processamento e à segurança dos Dados pessoais do cliente com relação ao GitHub AE (o "**Serviço online**"). O GitHub assume os compromissos descritos neste DPA perante todos os clientes que usam o Serviço online.

Na hipótese de conflitos ou inconsistências entre o DPA e quaisquer outros termos em contratos do Cliente com o GitHub ("Contrato"), o DPA prevalecerá. Para fins de clareza, consistente com a Cláusula 10 das Cláusulas contratuais padrão no Anexo 1, as Cláusulas contratuais padrão prevalecem sobre quaisquer outros termos no DPA.

## CONTRATO DE PROTEÇÃO DE DADOS DO GITHUB

### 1. Definições.

1.1 "**Leis de proteção de dados aplicáveis**" são determinadas leis, regulamentações, marcos regulatórios ou outras legislações relacionadas ao processamento e ao uso de Dados pessoais do Cliente, conforme aplicável ao uso dos Serviços online pelo Cliente, incluindo:

  a. O Regulamento Geral sobre a Proteção de Dados da União Europeia 2016/679 (“**GDPR**”), com quaisquer leis ou regulamentos nacionais equivalentes; e

  b. California Consumer Privacy Act de 2018, Código Civil da Califórnia §§1798.100 e seguintes ("**CCPA**").

1.2 Os termos “**Controlador**,” “**Titular de dados**,” “**Estados-membro**,” “**Dados pessoais**,” “**Violação de dados pessoais**,” “**Processamento**,” “**Processador**,” e “**Autoridade supervisora**” têm os significados atribuídos a eles nas Leis de Proteção de Dados aplicáveis. Em caso de conflito, os significados atribuídos no GDPR terão precedência.

1.3 O termo "**Dados pessoais do cliente**" significa quaisquer Dados pessoais dos quais o Cliente seja Controlador, independentemente de serem fornecidos pelo Cliente para processamento pelo GitHub ou gerados pelo GitHub durante a execução de suas obrigações previstas no Contrato. O termo inclui dados como informações de cobrança, endereços IP, endereços de email corporativos e todos os outros Dados pessoais dos quais o Cliente seja Controlador.

1.4 O termo "**Dados de repositório do cliente**" significa quaisquer dados ou informações que sejam carregados ou criados pelo Cliente em um dos respectivos Repositórios privados.

1.5 O termo "**Violação de dados**" significa uma Violação de dados pessoais ou qualquer violação confirmada ou razoavelmente suspeita dos Dados protegidos do Cliente.

1.6 “**Operações comerciais legítimas do GitHub**” consistem no seguinte, sendo cada item consequência do fornecimento do Serviço ao Cliente: (1) elaboração de relatórios e modelos de negócios internos; (2) combate a fraude, crime cibernético ou ataques cibernéticos que podem afetar o GitHub; (3) aprimoramento da funcionalidade principal de acessibilidade ou privacidade; e (4) relatórios financeiros e conformidade com obrigações legais.

1.7 O termo “**Usuário final**” significa os Usuários finais do Cliente, tais como funcionários, prestadores de serviço ou colaboradores.

1.8 "**Finalidades permitidas**" para processamento de dados são as finalidades limitadas e específicas para o fornecimento do Serviço online, conforme estabelecidas no Contrato e neste DPA, ou as finalidades para as quais um Titular de dados autorizou o uso de Dados pessoais do cliente.

1.9 "**Dados protegidos**" inclui Dados pessoais do cliente e Dados de repositório do cliente processados pelo GitHub em nome do Cliente segundo o Contrato.

1.10 O termo "**Dados confidenciais**" significa quaisquer Dados pessoais que revelem origem racial ou étnica; opiniões políticas, crenças religiosas ou filosóficas ou filiação sindical; processamento de dados genéticos ou biométricos para identificar uma pessoa física de maneira inequívoca; dados relativos à saúde, à orientação ou à vida sexual de uma pessoa física e dados relativos a infrações, condenações penais ou medidas de segurança.

## 2. Status e conformidade.

#### 2.1 Processamento de dados.
O GitHub atua como Processador em relação a quaisquer Dados pessoais do cliente que recebe em relação ao Contrato, exceto quando os Clientes atuam como Processadores (nesse caso, o GitHub é um subprocessador). O GitHub processará os Dados pessoais do cliente somente para as Finalidades permitidas de acordo com as instruções do Cliente, conforme o Contrato e outras comunicações escritas. Caso o GitHub não consiga cumprir as instruções do Cliente (por exemplo, devido a conflitos com as Leis de proteção de dados aplicáveis ou quando o processamento é exigido pelas Leis de proteção de dados ou por outros requisitos legais aplicáveis), o GitHub notificará o Cliente desde que permitido. Embora o GitHub processe todos os Dados pessoais do cliente nos Estados Unidos ou na União Europeia, subprocessadores do GitHub poderão processar dados fora dos Estados Unidos ou da União Europeia. Além disso, o GitHub atua como Processador dos Dados de repositório do cliente.

#### 2.2 Controladores de dados.
O Cliente é Controlador somente dos Dados pessoais do cliente que ele transfere diretamente ao GitHub ou por meio do uso do Serviço Online, exceto: (a) quando o Cliente atua como Processador de Dados pessoais do cliente (nesse caso, o GitHub é um subprocessador); ou (b) na medida em que o GitHub usa, ou de alguma forma processa, Dados pessoais do cliente para as Operações comerciais legítimas do GitHub. Nesse caso, o GitHub cumprirá com as obrigações de um controlador de dados independente conforme a GDPR as define para tal fim.

#### 2.3 Conformidade do GitHub; transferências de dados.
O GitHub cumprirá as Leis de proteção de dados aplicáveis em relação ao processamento de Dados pessoais.

Todas as transferências de dados pessoais do cliente para fora da União Europeia, do Espaço Econômico Europeu, do Reino Unido e da Suíça para fornecer o Serviço online serão regidas pelas Cláusulas contratuais padrão no Anexo 1 (Cláusulas contratuais padrão).

O GitHub cumprirá os requisitos do Espaço Econômico Europeu e da legislação suíça de proteção de dados referentes à coleta, ao uso, à transferência, à retenção e a outros tipos de processamento de Dados pessoais do Espaço Econômico Europeu, do Reino Unido e da Suíça. Todas as transferências de Dados pessoais a um terceiro país ou a uma organização internacional estarão sujeitas a proteções adequadas, conforme descrito no Artigo 46 do GDPR, e tais transferências e proteções serão documentadas de acordo com o Artigo 30(2) do GDPR.

Além disso, o GitHub está certificado com relação à Estrutura Privacy Shield UE-EUA e Suíça-EUA e os compromissos assim assumidos, embora o GitHub não empregue a Estrutura Privacy Shield UE-EUA como fundamento legal para as transferências de Dados pessoais à luz da decisão judicial do Tribunal de Justiça da UE no Processo C-311/18.  O GitHub concorda em notificar o Cliente se determinar que não pode mais cumprir sua obrigação de fornecer o mesmo nível de proteção exigido pelos princípios do Privacy Shield.

#### 2.4 Direitos do titular de dados; assistência com solicitações.
O GitHub disponibilizará ao Cliente, de modo consistente com a funcionalidade do Serviço online e com o papel do GitHub como processador de Dados pessoais do cliente, a possibilidade de atender a solicitações do titular de dados de exercer os respectivos direitos de acordo com as Leis de proteção de dados aplicáveis, como a GDPR e o CCPA. Se o GitHub receber uma solicitação de um titular de dados do Cliente para exercer um ou mais de seus direitos em relação ao Serviço online do qual o GitHub é processador ou subprocessador de dados, o GitHub encaminhará o titular de dados para fazer a solicitação diretamente ao cliente. O Cliente será responsável por responder a qualquer solicitação desse tipo, incluindo, quando necessário, usando a funcionalidade ou documentação fornecida pelo Serviço online. O GitHub deverá cumprir as solicitações razoáveis feitas pelo Cliente para auxiliar na resposta do Cliente quanto à solicitação do titular dos dados.

### 3. Proteção de dados.

#### 3.1 Limitação de finalidade.
O GitHub processará e comunicará os Dados protegidos apenas para as Finalidades permitidas, a menos que as Partes concordem, por escrito, com uma finalidade expandida.

#### 3.2 Qualidade e proporcionalidade dos dados.
O GitHub manterá os Dados pessoais do cliente precisos e atualizados, ou habilitará o Cliente a fazê-lo. O GitHub adotará medidas comercialmente razoáveis para garantir que os Dados protegidos coletados em nome do Cliente sejam adequados, relevantes e não sejam excessivos em relação às finalidades para as quais forem transferidos e processados. Em nenhuma hipótese o GitHub coletará intencionalmente Dados confidenciais em nome do Cliente. O Cliente concorda que o Serviço online não se destina ao armazenamento de Dados confidenciais. Se optar por carregar Dados confidenciais no Serviço online, o Cliente deverá cumprir o Artigo 9 da GDPR ou disposições equivalentes nas Leis de proteção de dados aplicáveis.

#### 3.3 Retenção e exclusão de dados.
Mediante solicitação razoável do Cliente, a menos que seja proibido por lei, o GitHub devolverá, destruirá ou removerá a identificação de todos os Dados pessoais do cliente e dados relacionados em todos os locais onde estejam armazenados, depois que não forem mais necessários para as Finalidades permitidas, no prazo de trinta dias após a solicitação. O GitHub pode reter Dados pessoais do cliente e dados relacionados, na medida exigida pelas Leis de proteção de dados aplicáveis e apenas durante o período estipulado por elas, desde que o GitHub assegure que os Dados pessoais do cliente serão processados somente conforme necessário para a finalidade especificada nas Leis de proteção de dados aplicáveis e que os Dados pessoais do cliente permaneçam protegidos por essas leis.

#### 3.4 Processamento de dados.
No que diz respeito ao processamento de Dados protegidos do cliente, o GitHub fornece as seguintes informações exigidas pelo Artigo 28(3) do GDPR:

a.  *O assunto e a duração do processamento* de Dados pessoais do cliente estão definidos no Contrato e no DPA.

b.  *A natureza e a finalidade do processamento* de Dados pessoais do cliente estão descritos na Seção 3.1 do DPA.

c.  *Os tipos de Dados pessoais do cliente a serem processados* são descritos no Contrato e incluem os Dados pessoais do cliente; ou qualquer tipo de Dados pessoais que o Cliente escolha incluir nos Dados pessoais do cliente. O Cliente pode optar por fornecer ao GitHub Dados pessoais do cliente adicionais; por exemplo, nas configurações de perfil do Cliente ou carregando Dados pessoais do cliente nos repositórios do GitHub.

d.   *As categorias de Titular de dados a que estão relacionados os Dados pessoais do cliente* são o próprio Cliente e os respectivos Usuários finais.

e.   *As obrigações e os direitos do Cliente* estão previstos no Contrato e no DPA.

### 4. Obrigações de Segurança e Auditoria.

#### 4.1 Medidas de segurança técnicas e organizacionais.
Levando em consideração o que há de mais moderno, os custos de implementação e a natureza, o escopo, o contexto e as finalidades do processamento, bem como o risco de probabilidade e gravidade variáveis para os direitos e as liberdades fundamentais das pessoas físicas, o GitHub implementará medidas técnicas e organizacionais para garantir um nível de segurança adequado contra os riscos (como destruição acidental ou ilegal, perda, alteração, divulgação ou acesso não autorizado) que se apresentam ao processar Dados protegidos. O GitHub monitorará regularmente o cumprimento dessas medidas e continuará tomando as providências apropriadas ao longo da vigência do Contrato. Consulte a Seção 1.1 do Anexo de segurança (abaixo) relativa às responsabilidades do GitHub do que diz respeito às proteções de segurança.

#### 4.2 Resposta a incidentes e notificação de violação.
O GitHub cumprirá as obrigações de Segurança da informação no Anexo de segurança e nas Leis de proteção de dados aplicáveis, incluindo as obrigações sobre notificação de Violação de dados. Consulte a Seção 1.2 do Anexo de segurança para conhecer as responsabilidades do GitHub relacionadas à notificação e à resposta a Violações de dados.

#### 4.3 Equipe do GitHub.
O GitHub declara e garante que tomará medidas razoáveis para assegurar que todos os funcionários do GitHub que processarem Dados protegidos tenham concordado em manter a confidencialidade dos Dados protegidos e tenham recebido treinamento adequado sobre o cumprimento do DPA e das Leis de proteção de dados aplicáveis.

#### 4.4 Registros.
O GitHub manterá registros por escrito completos, precisos e atualizados de todas as categorias de atividades de processamento realizadas em nome do Cliente que contenham as informações exigidas de acordo com as Leis de proteção de dados aplicáveis. Desde que a assistência não arrisque a segurança do GitHub nem os direitos à privacidade de Titulares de dados individuais, o GitHub disponibilizará esses registros ao Cliente mediante solicitação conforme a necessidade, por exemplo, para ajudar o Cliente a demonstrar a conformidade com as Leis de proteção de dados aplicáveis. Para saber mais sobre os requisitos do GitHub de oferecer assistência no caso de um incidente de segurança, consulte a Seção 1.2 do Anexo de segurança.

#### 4.5 Relatório de conformidade.
O GitHub fornecerá relatórios de conformidade de segurança de acordo com a Seção 2.3 do Anexo de segurança. O Cliente concorda que quaisquer direitos a informações e auditoria concedidos pelas Leis de proteção de dados aplicáveis (inclusive, quando aplicável, o Artigo 28(3)(h) da GDPR) serão satisfeitos por esses relatórios de conformidade e só serão efetivos na medida em que o fornecimento de um relatório de conformidade pelo GitHub não fornecer informações suficientes ou na medida em que o Cliente precisar responder a uma auditoria de uma Autoridade supervisora ou regulatória. A Seção 3.1 do Anexo de segurança descreve as responsabilidades das Partes em relação a uma auditoria da Autoridade supervisora ou regulatória.

#### 4.6 Assistência.
O GitHub fornecerá a assistência cabível ao Cliente para questões como avaliações de impacto sobre a privacidade de dados, solicitações de direitos do Titular de dados, consultas com Autoridades supervisoras e outros assuntos semelhantes; em todos os casos, exclusivamente em relação ao processamento de Dados pessoais do cliente e levando em consideração a natureza do processamento.

### 5. Uso e divulgação de dados protegidos.
Nenhum uso para fins de marketing e CCPA. O GitHub não usará os Dados protegidos para fins de publicidade de conteúdo de terceiros. Se processar Dados pessoais do cliente dentro do escopo do CCPA, o GitHub não reterá, usará nem divulgará esses dados para nenhuma finalidade que não sejam aquelas definidas no DPA e permitidas pelo CCPA, incluindo qualquer isenção de "venda". O GitHub não venderá nenhum dado desse tipo sob nenhuma circunstância.

### 6. Subprocessamento e transferência posterior.

#### 6.1 Proteção de dados.
O GitHub é responsável por transferências posteriores de Dados protegidos para seus subprocessadores, como o processador de pagamentos terceirizado. Na hipótese de o GitHub transferir Dados protegidos para um subprocessador terceirizado ou de o GitHub instalar, usar ou permitir que terceiros ou serviços de terceiros processem Dados protegidos em nome do GitHub, o GitHub garantirá que o subprocessador terceirizado seja vinculado a um contrato por escrito exigindo que ele forneça, pelo menos, o mesmo nível de confidencialidade, proteção de privacidade e segurança exigido do GitHub por este DPA e pelas Leis de proteção de dados aplicáveis.

#### 6.2 Aceitação de subprocessadores do GitHub.
O Cliente autoriza o GitHub e seus subprocessadores a designar e utilizar subprocessadores de acordo com a Seção 6 e o Contrato.

#### 6.3 Consentimento geral para subprocessamento posterior.
O Cliente fornece consentimento geral para que o GitHub utilize subprocessadores posteriores, mediante a conformidade do GitHub com os seguintes requisitos:

a.  Qualquer subprocessador posterior deve concordar por escrito em somente processar dados em países que a Comissão Europeia tenha declarado como tendo um nível "adequado" de proteção, ou somente processar dados segundo termos equivalentes às Cláusulas contratuais padrão, ou de acordo com uma autorização das Regras corporativas vinculantes concedida pelas autoridades europeias de proteção de dados competentes ou conforme certificação válida do Privacy Shield UE-EUA e Suíça-EUA; e

b.  O GitHub restringirá o acesso do subprocessador posterior aos Dados pessoais do cliente apenas ao que for estritamente necessário para executar seus serviços, e o GitHub proibirá o subprocessador de processar os Dados pessoais do cliente para qualquer outra finalidade.

#### 6.4 Divulgação de contratos do subprocessador.
O GitHub mantém uma lista de subprocessadores posteriores que utilizou para processar Dados pessoais do cliente em seu site, incluindo as categorias de Dados pessoais do cliente processados, uma descrição do tipo de processamento que o subprocessador executa e a localização do processamento. Mediante solicitação por escrito do Cliente, o GitHub fornecerá a ele esta lista de subprocessadores e os termos segundo os quais eles processarão os Dados pessoais do cliente. De acordo com as restrições de confidencialidade do subprocessador, o GitHub poderá remover informações confidenciais ou comercialmente sigilosas antes de fornecer a lista e os termos ao Cliente. Caso o GitHub não possa divulgar informações confidenciais ou sigilosas ao Cliente, as Partes concordam que o GitHub fornecerá todas as informações razoavelmente possíveis relacionadas aos contratos de subprocessamento.

#### 6.5 Objeção a subprocessadores.
O GitHub fornecerá aviso por escrito trinta dias antes da adição ou remoção de qualquer subprocessador, incluindo as categorias listadas na Seção 6.4, anunciando mudanças em seu site. Se tiver uma objeção razoável à utilização de um novo subprocessador pelo GitHub, o Cliente deverá notificar o GitHub prontamente por escrito. Sempre que possível, o GitHub envidará esforços comercialmente razoáveis para fornecer uma solução alternativa aos Serviços online para evitar o processamento de dados pelo subprocessador questionável. Caso o GitHub não consiga oferecer uma solução alternativa e as Partes não resolvam o conflito em noventa dias, o Cliente poderá rescindir o Contrato.

### 7. Rescisão.

#### 7.1 Suspensão.
Na hipótese de o GitHub violar suas obrigações de manter um nível adequado de segurança ou proteção da privacidade, o Cliente poderá suspender temporariamente a transferência de todos os Dados pessoais do cliente ou proibir a coleta e o processamento desses dados em nome do Cliente até que a violação seja reparada ou o Contrato seja rescindido.

#### 7.2 Rescisão com justa causa.
Além dos direitos de rescisão que o Cliente tem previstos no Contrato, o Cliente poderá rescindir o Contrato sem prejuízo de qualquer outra reivindicação, segundo a lei ou em equidade, caso:

a.  o GitHub notifique o cliente de que não poderá mais cumprir suas obrigações de privacidade;

b.  a transferência, a coleta ou o processamento de todos os Dados pessoais do cliente tiver sido temporariamente suspenso por mais de um mês, de acordo com a Seção 7.1;

c.  o GitHub viole, de modo substancial ou persistente, quaisquer garantias ou declarações feitas neste DPA;

d.  o GitHub não esteja mais em atividade, seja dissolvido, entre em liquidação ou tenha uma ordem de liquidação feita em seu nome; ou

e.  o Cliente se oponha a um subprocessador de acordo com a Seção 6.5 e o GitHub não tenha conseguido fornecer uma solução alternativa em até noventa dias.

#### 7.3 Violação.
A não observância das disposições materiais deste DPA é considerada uma violação material de acordo com o Contrato.

#### 7.4 Não cumprimento.
Na hipótese de mudanças na lei ou na regulamentação tornarem o cumprimento do DPA impossível ou comercialmente impraticável, as Partes poderão renegociar o DPA de boa-fé. Se a renegociação não remediar a impossibilidade ou se as Partes não conseguirem chegar a um acordo, elas poderão rescindir o Contrato após trinta dias.

#### 7.5 Notificação.
Caso determine que não pode mais cumprir as obrigações de privacidade previstas no DPA, o GitHub notificará o Cliente por escrito imediatamente.

#### 7.6 Modificações.
O GitHub poderá modificar o DPA ocasionalmente, conforme exigido pelas Leis de proteção de dados aplicáveis, com aviso prévio de trinta dias ao Cliente.

#### 7.7 Requisitos de rescisão.
Mediante uma rescisão, o GitHub deverá:

a.  tomar medidas razoáveis e apropriadas para interromper o processamento dos Dados pessoais do cliente;

b.  dentro de noventa dias após a rescisão, excluir ou remover a identificação dos Dados pessoais do cliente que o GitHub armazena em nome do Cliente, de acordo com a Seção 3.3; e

c.  fornecer ao Cliente garantia razoável de que o GitHub cumpriu suas obrigações de acordo com a Seção 7.7.

### 8. Responsabilidade pelo processamento de dados.

#### 8.1 Limitações.
Exceto quando limitado pelas Leis de proteção de dados aplicáveis, quaisquer reivindicações referentes ao DPA estarão sujeitas aos termos do Contrato no que tange Limitações de responsabilidade.

## Anexo 1 – Cláusulas contratuais padrão (Processadores)
A execução do Contrato pelo Cliente inclui a execução deste Anexo 1 ao Adendo de Proteção de Dados do GitHub, ratificado pela GitHub, Inc.

Em países nos quais a aprovação regulatória é necessária para uso das Cláusulas Contratuais Padrão, elas não podem se basear na Comissão Europeia 2010/87/EU (de fevereiro de 2010) para legitimar a exportação dos dados do país, a menos que o Cliente tenha a aprovação regulatória necessária.

Para os fins do Artigo 46 (2) do Regulamento Geral sobre a Proteção de Dados (EU 2016/679) para a transferência de dados pessoais a processadores estabelecidos em países terceiros que não garantem um nível adequado de proteção dos dados, o Cliente (como o exportador dos dados) e o GitHub (como o importador dos dados, cuja assinatura é exibida a seguir), individualmente uma “parte”, coletivamente “as partes”, concordaram com as seguintes Cláusulas contratuais (“Cláusulas” ou “Cláusulas contratuais padrão”) a fim de fornecer garantias adequadas relativas à proteção de direitos fundamentais e à privacidade e liberdade de indivíduos para a transferência realizada pelo exportador ao importador dos dados pessoais especificados no Apêndice 1.

#### Cláusula 1: Definições
(a) os termos "dados pessoais", "categorias especiais de dados", "processo/processamento", "controlador", "processador", "titular dos dados" e "autoridade supervisora" terão o mesmo significado que no Regulamento Geral sobre a Proteção de Dados (EU 2016/679) referente à proteção de indivíduos no que se refere ao processamento de dados pessoais e à livre circulação desses dados;

(b) o termo “exportador de dados” significa o controlador que transfere os dados pessoais;

(c) o termo "importador de dados" significa o processador que aceita receber do exportador de dados os dados pessoais destinados a processamento em seu nome após a transferência, de acordo com suas instruções e os termos das Cláusulas, e que não está sujeito a um sistema de um país terceiro, garantindo proteção adequada no âmbito do Artigo 45(2) do Regulamento Geral sobre a Proteção de Dados (EU 2016/679);

(d) o termo “subprocessador” significa qualquer processador empregado pelo importador de dados ou por qualquer outro subprocessador do importador de dados que aceita receber do importador de dados ou de qualquer outro subprocessador do importador de dados os dados pessoais exclusivamente destinados a atividades de processamento a serem realizadas em nome do exportador de dados depois da transferência, de acordo com suas instruções, com os termos das Cláusulas e com os termos do subcontrato escrito;

(e) o termo “lei de proteção de dados aplicável” significa a legislação que protege os direitos fundamentais e a liberdade do indivíduo e, especialmente, seu direito à privacidade com relação ao processamento de dados pessoais aplicáveis a um controlador de dados no Estado-membro no qual o exportador de dados está estabelecido;

(f) o termo “medidas de segurança técnicas e organizacionais” significa as medidas que visam proteger os dados pessoais contra destruição acidental ou ilícita ou perda acidental, alteração, divulgação ou acesso não autorizados, especialmente quando o processamento envolve a transmissão de dados por uma rede, e contra todas as formas ilícitas de processamento.

Os detalhes da transferência e em particular as categorias especiais de dados pessoais, onde aplicável, são especificadas no Apêndice 1 abaixo que constituem uma parte integral das Cláusulas.

#### Cláusula 3: Cláusula de terceiro beneficiário

O GitHub também informa seu pessoal sobre as possíveis consequências da violação de regras e procedimentos de segurança. O GitHub usará apenas dados anônimos para treinamento.

#### 1.2 Gestão de incidentes de segurança.
O GitHub fornecerá um programa de gestão de incidentes de segurança para o Serviço online da seguinte forma:

a.  Escalonamento e disponibilidade de segurança. O GitHub manterá os processos apropriados de escalonamento e contato de segurança 24 horas por dia, 7 dias por semana, para garantir que clientes e funcionários possam enviar problemas para a equipe de Segurança do GitHub.

b.  Resposta a incidentes. Se o GitHub tomar conhecimento de uma violação de segurança que leve a destruição acidental ou ilegal, perda, alteração, divulgação não autorizada ou acesso a Dados do cliente ou Dados pessoais (considerados individualmente como "Incidente de segurança"), o GitHub vai, prontamente e sem demora indevida, (1) notificar o Cliente sobre o Incidente de segurança; (2) investigar o Incidente de segurança e fornecer ao Cliente informações detalhadas sobre o incidente; (3) tomar medidas razoáveis para atenuar os efeitos e minimizar os danos resultantes do Incidente de segurança.

c.  Notificação. As notificações de Incidentes de segurança serão feitas a um ou mais administradores do Cliente por qualquer meio que o GitHub selecionar. É responsabilidade exclusiva do Cliente garantir que os administradores do Cliente monitorem e respondam a quaisquer notificações. O Cliente é o único responsável pelo cumprimento de suas obrigações sob as leis de notificação de incidentes aplicáveis ao Cliente e pelo cumprimento de quaisquer obrigações de notificação de terceiros relacionadas a qualquer Incidente de Segurança.

d.  Assistência razoável. O GitHub envidará esforços comercialmente razoáveis para ajudar o Cliente a cumprir as respectivas obrigações segundo as leis ou a regulamentação aplicável de notificar os titulares de dados e a autoridade supervisora relevante sobre o Incidente de segurança.

a. O Cliente deve garantir o uso de um terceiro independente (ou seja, o regulador ou representante do regulador) e que o acesso do Cliente a resultados e dados não relevantes para ele seja impedido.

b. A notificação de tal auditoria deve ser feita por escrito ao GitHub em tempo hábil, mediante notificação do regulador e de maneira a permitir que o pessoal adequado seja disponibilizado para auxiliar. Quando os reguladores não avisarem com antecedência ao Cliente sobre a auditoria ou a investigação, o GitHub responderá no momento adequado, conforme exigido pelos reguladores.

c. Qualquer auditor terceirizado deve divulgar ao GitHub os resultados e as ações recomendadas, quando permitido pelo regulador.

d. Em caso de auditoria regulatória, o acesso será permitido apenas durante o horário comercial normal (Hora do Pacífico).

e. Na medida permitida por lei, o Cliente deve manter a confidencialidade de qualquer informação obtida por meio de qualquer auditoria do GitHub que, por natureza, deva ser confidencial.
